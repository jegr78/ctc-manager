### Tech-Debt Findings

**W-A — Graphics services on legacy `calculateStandings(seasonId)` bridge** (REQ: SVC-02)
5 callers in `org.ctc.admin.service`: `AbstractMatchdayGraphicService:47`, `TeamCardService:52`, `OverlayGraphicService:58`, `SettingsGraphicService:67`, `LineupGraphicService:70`. Bridge delegates to `findRegularPhase(seasonId)` — no runtime regression. Original `@Deprecated` was re-classified as permanent convenience overload. Removal would require migrating these 5 callers first.

**W-B — No UI affordance for per-group matchday generation** (REQ: UI-03 / QUAL-02)
`SeasonController.generateMatchdays:251` hardcodes `groupId = null`. GROUPS-layout users cannot generate matchdays bound to a group via the admin UI. Phase 61 QUAL-02 E2E test had to use repositories directly. Acceptable for v1.9 release; future phase should add per-group generator route.

### Observations

**O-1** — Phase 60 has no `60-VERIFICATION.md` (covered above)

**O-2** — Phase 62 not in REQUIREMENTS.md traceability table. Currently borrows UI-02 / UI-05 / UI-07 IDs. Recommend adding `SITE-01..03` IDs covering "public-site phase + group awareness, PLAYOFF reachability, byte-identical LEAGUE-only baseline" so Phase 62 has unambiguous ownership.

**O-3** — Legacy `POST /admin/playoffs/{id}/add-season` returns HTTP 410 Gone (with `admin/error` template + meaningful "endpoint retired in v1.9" message), not 5xx. Better UX than the spec's described behavior.
