### Phase 75: Replace-All Transaction + JPA Auditing Bypass + Live MariaDB UAT

**Goal**: The riskiest phase. A single `@Transactional` boundary wipes all 22 operative tables in FK-reverse order via `EntityManager.createNativeQuery("DELETE FROM ...")` (TRUNCATE auto-commits on MariaDB → forbidden), drops the L1 cache via `em.flush() + em.clear()`, then bulk-restores via `JdbcTemplate.batchUpdate` (bypassing `AuditingEntityListener` so imported `created_at`/`updated_at` survive verbatim). The `Team.parentTeam` self-FK is decoupled in a pre-step (`UPDATE teams SET parent_team_id = NULL`). Upload-tree restore is post-commit (file-system mutations are NOT in the JPA transaction) via stage-and-rename, leaving the previous tree at `data/.import-backups/<ts>/uploads-old/` for 24 h manual recovery. A `BackupImportRollbackIT` injects an exception at 50 % of the restore loop and asserts the DB is in pre-import state with `success=false` audit row. Live UAT against the Saison-2023 fixture on local MariaDB confirms standings + driver-ranking + phase-breakdowns are byte-identical after a full export → wipe → import cycle.

**Depends on**: Phase 74 (needs the schema-version-validated, hardened ZIP reader from preview before write semantics are enabled)

**Requirements**: IMPORT-05, IMPORT-06, IMPORT-07, QUAL-03

**Success Criteria** (what must be TRUE):

  2. Imported entities preserve their original `created_at` / `updated_at` / `version` values verbatim (an IT exporting a row with `created_at=2024-01-01`, wiping, and importing asserts the value is still `2024-01-01` after re-read)

  4. The previous `uploads/` directory tree is preserved at `data/.import-backups/<ts>/uploads-old/` after a successful import; the new tree is in place atomically (no half-written files visible to readers between the rename steps)
  5. Live MariaDB UAT (Saison-2023 driver fixture) confirms via the public site that standings, driver ranking, and phase breakdowns are visually identical after export → wipe → import

**Goal**: Defense-in-depth around the Replace-All path. `ImportLockService` (`@Service @Scope("singleton")`) holds a `ReentrantLock`; a second concurrent import attempt is rejected with HTTP 409 + Flash.
