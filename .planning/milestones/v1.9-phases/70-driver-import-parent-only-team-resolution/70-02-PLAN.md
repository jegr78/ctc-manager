<objective>
Phase 70 D-09 says: the Group column header, the per-row Group cells (3-branch span block), the warning box, the `showGroupColumn` model attribute, and the controller's page-wide GROUPS-detection are all to be removed from the driver-import preview surface. D-08 says: Group display in the driver-import context generally makes no sense — Group assignment is per-match (RaceLineup), not per-season. This plan deletes that UX and the test cases that pin it in place.

Purpose: Group display at the import preview level was Phase-66-D-05 + D-06 surface scaffolding paired with the now-inverted resolver behavior. With Phase 70 D-05 making the resolver parent-precedence-only, Group display has no source of truth at the preview stage and is misleading.

Output: Slimmed `DriverSheetImportController`, slimmed `driver-import-preview.html` (5 buckets without Group column + no warning box), and slimmed `DriverSheetImportControllerTest` (the two GROUPS-related preview tests removed).

**Wave-1 parallelism:** This plan touches `DriverSheetImportController.java`, `driver-import-preview.html`, and `DriverSheetImportControllerTest.java` — all disjoint from `DriverSheetImportService.java` (Plan 70-01). Both plans land in Wave 1. Plan 70-03 reconciles the service-side test suite + final verify in Wave 2.

**Important coupling note:** `DriverSheetImportController.java` currently imports `org.ctc.domain.model.PhaseLayout` + `org.ctc.domain.model.PhaseType` solely for the `showGroupColumn` calculation. Other controllers (e.g. `SeasonController`, `StandingsController`, `SeasonPhaseController`) also use `showGroupColumn` for their own standings views — those are out of scope (Phase-70-CONTEXT explicitly lists them as untouched). This plan only removes `showGroupColumn` from the **driver-import preview** model, never from standings.
</objective>
