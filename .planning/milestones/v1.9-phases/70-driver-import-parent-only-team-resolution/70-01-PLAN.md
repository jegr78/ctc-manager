<objective>
Phase 70 corrects a domain-model violation introduced by Phase 66 D-04. SeasonDriver.team must always reference the parent team — sub-team assignment happens per-match via RaceLineup, never per-phase. This plan rewrites the entire group-resolution surface inside `DriverSheetImportService.java`: parent-precedence resolver, no PhaseTeam lookup in the preview path, no TEAM_NOT_IN_REGULAR_PHASE warning, slimmed-down records.

Purpose: The current `resolveTeamByShortName(shortName, regularPhase)` returns the sub-team if it has a PhaseTeam in the target REGULAR phase. Live UAT (Saison 2023, parent MRL + sub-teams MRL 1/MRL 2 each in different groups) showed this assigns drivers to sub-teams at the season level — a per-match concern leaking into per-season state. Inverting the default makes sheet-imported `SeasonDriver.team` always parent (or solo team), which matches the user-confirmed domain model (CONTEXT D-01..D-04).

Output: Refactored `DriverSheetImportService.java`. **Tests are NOT updated in this plan** — that is Plan 70-03's responsibility. Plan 70-01 will leave the test suite RED on tests #15, #16, #17, #18, #19, #20, #23, #24 in `DriverSheetImportServiceTest` and on tests #4, #5, #6 in `DriverSheetImportServiceIT` (records changed). UI-related tests stay GREEN until Plan 70-02 ships. We do not run a full `./mvnw verify` mid-phase (D-21).

    ```java
    /**
     * Resolves a team by shortName with parent-precedence on multi-match.
     * <p>
     * The {@code teams.short_name} column is intentionally non-unique — a parent team and
     * one of its sub-teams may share the same shortName (e.g. parent {@code ZFS} + sub
     * {@code ZFS}). Per Phase 70 (D-01..D-05) the import always assigns the parent at the
     * season level; sub-team variation is per-match (RaceLineup), not per-season.
     * <ol>
     *   <li><b>0 matches:</b> empty — caller emits {@code UNKNOWN_TEAM_CODE}.</li>
     *   <li><b>1 match:</b> return it (parent or solo-sub with its own unique shortName, both legitimate).</li>
     *   <li><b>N matches:</b> return the first {@code parentTeam == null} candidate.
     *       If no candidate is a parent (data-integrity edge), log WARN and return the first deterministically.</li>
     * </ol>
     * Inverts Phase 66 D-04 — see {@code 70-CONTEXT.md} D-05.
     *
     * @param shortName trimmed team short code from the sheet
     * @return the resolved team (parent precedence on multi-match), or empty if no team matches
     */
    private Optional<Team> resolveTeamByShortName(String shortName) {
        List<Team> matches = teamRepository.findAllByShortName(shortName);
        if (matches.isEmpty()) {
            return Optional.empty();
        }
        if (matches.size() == 1) {
            return Optional.of(matches.get(0));
        }
        Optional<Team> parent = matches.stream()
                .filter(t -> t.getParentTeam() == null)
                .findFirst();
        if (parent.isPresent()) {
            return parent;
        }
        log.warn("Multiple teams share shortName '{}' with no parent — picking first deterministically (data-integrity issue)", shortName);
        return Optional.of(matches.get(0));
    }
    ```
