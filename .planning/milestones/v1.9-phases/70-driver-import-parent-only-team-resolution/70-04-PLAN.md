Counter semantics post-fix (CRITICAL):
- `incrementNewDrivers()` MUST fire only inside `.orElseGet(...)` — i.e.
  only when a brand-new Driver row is actually persisted.
- When `findByPsnId(psnId)` returns a present Optional (recovery path:
  cache miss but DB has the driver), the existing Driver is reused; the
  counter is NOT incremented because no new Driver was created.
- `incrementNewAssignments()` is NOT inside the lambda — it remains on
  the post-`computeIfAbsent` line and fires for every NEW_DRIVER row
  regardless of whether the Driver is brand-new or recovered. This
  matches the WR-01 contract and the existing IT Test #4 expectations
  (`getNewAssignmentsCount() == 1` for one NEW_DRIVER row, regardless of
  Driver-side recovery).
