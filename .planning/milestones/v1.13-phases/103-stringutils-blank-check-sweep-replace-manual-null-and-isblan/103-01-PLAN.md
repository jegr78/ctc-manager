From `org.springframework.util.StringUtils` (Spring Framework 7.x core, on classpath via Spring Boot 4):

```java
package org.springframework.util;

public abstract class StringUtils {
    public static boolean hasText(@Nullable CharSequence str);
    public static boolean hasText(@Nullable String str);
    // ... (other methods irrelevant to this phase)
}
```

Behavioral equivalence (the only contract the executor needs):

| Pattern | Replacement | Equivalent because |
|---------|-------------|--------------------|
| `s != null && !s.isBlank()` | `hasText(s)` | `hasText(null)` returns `false`; `hasText("")` returns `false`; `hasText("   ")` returns `false` (whitespace-only is "blank"); `hasText("x")` returns `true`. Identical truth table to `s != null && !s.isBlank()` for every `String s`. |
| `s == null \|\| s.isBlank()` | `!hasText(s)` | De Morgan; same truth table. |

Static-import form (D-03, NON-NEGOTIABLE):

```java
import static org.springframework.util.StringUtils.hasText;
```

The non-static form `import org.springframework.util.StringUtils;` is **forbidden in the same file** (D-03b). The validation oracle (Task 9) will NOT detect a stray non-static import directly, but the executor MUST grep for it before commit (see Task 9 acceptance).
