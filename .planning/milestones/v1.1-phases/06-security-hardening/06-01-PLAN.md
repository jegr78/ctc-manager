    Implement two private validation methods in FileStorageService and wire them into the public methods.

    **1. SSRF hostname validation (per D-01, D-02, D-03 -- blocklist, string-based, no allowlist):**

    Add a private method `validateHostname(String sourceUrl)` that:
    - Extracts hostname from the URL using `java.net.URI.create(sourceUrl).getHost()`
    - Converts hostname to lowercase
    - Rejects if hostname is null
    - Rejects if hostname equals "localhost" or equals "[::1]"
    - Rejects if hostname matches IPv4 loopback: starts with "127."
    - Rejects if hostname matches private ranges: starts with "10.", starts with "192.168."
    - Rejects if hostname matches 172.16-31.x: starts with "172." and second octet is 16-31 (parse second octet as int, check >= 16 && <= 31)
    - Rejects if hostname matches link-local: starts with "169.254."
    - Throws `IllegalArgumentException("URL hostname blocked: " + hostname)` on rejection
    - Logs: `log.warn("Blocked SSRF attempt to internal host: {}", hostname)`

    Wire into `storeFromUrl()`: Call `validateHostname(sourceUrl)` immediately after the existing HTTPS check (after line 87).

    **2. Path traversal protection (per D-04, D-05 -- normalize+startsWith on all 3 writing methods):**

    Add a private method `validatePathWithinUploadDir(Path target)` that:
    - Normalizes: `Path normalized = target.toAbsolutePath().normalize()`
    - Checks: `if (!normalized.startsWith(uploadDir))` throw `IllegalArgumentException("Path traversal detected: " + target)`
    - Logs: `log.warn("Attempted path traversal: {}", target)`

    Wire into each writing method:
    - `store()`: After `Path target = raceDir.resolve(filename);` (line 40), call `validatePathWithinUploadDir(target);`
    - `storeImage()`: After `Path target = dir.resolve(safeName);` (line 104), call `validatePathWithinUploadDir(target);`
    - `storeFromUrl()`: After `Path target = dir.resolve(safeName);` (line 91), call `validatePathWithinUploadDir(target);`

    Also validate the directory itself in storeFromUrl to catch traversal via subDir parameter:
    - After `Path dir = uploadDir.resolve(subDir).resolve(entityId.toString());` (line 88), call `validatePathWithinUploadDir(dir);`

    Run full test suite: `./mvnw test -pl . -Dtest=FileStorageServiceTest`
