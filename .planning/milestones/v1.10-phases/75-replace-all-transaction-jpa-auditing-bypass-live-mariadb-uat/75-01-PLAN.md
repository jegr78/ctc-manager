The EntityRestorer SPI is intentionally collapsed from CONTEXT D-05's three-method shape to one method.
Reason: TeamRestorer (D-06) needs two SQL strings + two setters; a tableName()+insertSql()+setter() triple cannot model that.
The orchestrator (Plan 06) calls `restorer.restore(batch, jdbcTemplate)`; per-Restorer unit tests assert on the SQL emitted via a spy JdbcTemplate.

// EntityRestorer.java target shape (no implementation in this plan, just the contract):
package org.ctc.backup.restore;
import com.fasterxml.jackson.databind.JsonNode;
import org.springframework.jdbc.core.JdbcTemplate;
import java.util.List;
public interface EntityRestorer {
    String tableName();                          // snake_case; key into BackupSchema.getExportOrder()
    void restore(List<JsonNode> rows, JdbcTemplate jdbcTemplate);
}
