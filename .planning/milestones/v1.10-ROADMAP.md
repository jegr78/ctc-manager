## Overview

Two-track milestone: **(A)** Platform hygiene — Spring Boot 4.0.5 → 4.0.6 plus pinned Thymeleaf 3.1.5.RELEASE (CVE-2026-40478 SpEL canonicalization hardening), with controller-side `pageTitle` model attributes replacing fragment-parameter ternaries in `th:replace`/`th:insert`/`th:include` (~80 admin + site templates audited, build-guard installed via `exec-maven-plugin` to fence regression). **(B)** New admin **Backup Export / Import** feature — a streamed ZIP backup (`manifest.json` + per-entity JSON files under `data/` + entity-referenced `uploads/` mirror) and a replace-all import with mandatory preview, schema-version gate, ZIP-Slip + ZipBomb defenses, single-transaction wipe + restore via `JdbcTemplate.batchUpdate` (bypassing `AuditingEntityListener`), post-commit upload-tree stage-and-rename, persistent yellow read-only banner + `@ControllerAdvice` write-rejection + auto-backup-before-import safety net, full H2 + MariaDB round-trip coverage, and operator runbook. Closed by a code-cleanup + test-performance + Surefire/Failsafe forkCount sweep across the Phase 72-76 backup code. Side-quest: Phase 78 surgical Dockerfile `-noble` pin to repair the silently broken release workflow (`playwright install chromium` failing on rotated Ubuntu 26.04 base).

### Phase 73: Backup Export — Jackson MixIns + Streaming ZIP Endpoint

### Phase 74: Backup Import Preview + ZIP Hardening + Multipart Config + Schema-Version Gate

**Goal**: A no-write import path that reads the manifest, validates the schema version, and renders a per-table wipe + restore preview — without ever touching the database. ZIP-Slip and ZipBomb defenses harden the multipart upload (per-entry 50 MB / total 500 MB / 50.000 entries cap; `startsWith(uploadDir.toRealPath())` traversal check reusing the v1.1 SECU-02 defense). ### Phase 75: Replace-All Transaction + JPA Auditing Bypass + Live MariaDB UAT

**Goal**: The riskiest phase. A single `@Transactional` boundary wipes all 24 operative tables in FK-reverse order via `EntityManager.createNativeQuery("DELETE FROM ...")` (TRUNCATE auto-commits on MariaDB → forbidden), drops the L1 cache via `em.flush() + em.clear()`, then bulk-restores via `JdbcTemplate.batchUpdate` (bypassing `AuditingEntityListener` so imported `created_at`/`updated_at` survive verbatim). The `Team.parentTeam` self-FK is decoupled in a pre-step. Upload-tree restore is post-commit (file-system mutations are NOT in the JPA transaction) via stage-and-rename, leaving the previous tree at `data/.import-backups/<ts>/uploads-old/` for 24 h manual recovery. A `BackupImportRollbackIT` injects an exception at 50 % of the restore loop and asserts the DB is in pre-import state with `success=false` audit row. Live UAT against the Saison-2023 fixture on local MariaDB confirms standings + driver-ranking + phase-breakdowns are byte-identical after a full export → wipe → import cycle.

### Phase 76: Operational Hardening — Import Lock + Read-Only Banner + Auto-Backup-Before-Import

**Goal**: Defense-in-depth around the Replace-All path. `ImportLockService` (`@Service @Scope("singleton")`) holds a `ReentrantLock`; a second concurrent import attempt is rejected with HTTP 409 + Flash. While the lock is held, a yellow read-only banner renders on every admin page (English: "Import running — write access temporarily locked"); a `@ControllerAdvice` filter rejects any non-import POST under `/admin/**` with HTTP 503. Before the wipe transaction begins, `BackupImportService` synchronously triggers an auto-export to `data/.import-backups/<ts>/auto-backup-before-import.zip` — if the auto-export fails, the import aborts before any DB mutation.

## Milestone Summary

**Decimal Phases:** None.

**Key Decisions** (full list under PROJECT.md "Key Decisions"):

- `data_import_audit` permanently out of export scope; enforced structurally via `BackupSchema` package-name filter `org.ctc.domain.model.*` — any entity under `org.ctc.backup.*` is excluded with no opt-in / marker / developer memory required (Phase 72 D-15).
- Schema versioning is a monotonic integer, not a semver; importers refuse mismatched ZIPs BEFORE any DB write (Phase 72, GAP-2).
- `EXPORT_ORDER` is JPA-Metamodel-generated via Kahn's algorithm over `org.ctc.domain.model.*`; `Team.parentTeam` self-FK detected and skipped; hand-rolled FK orderings forbidden (Phase 72, GAP-5).
- 24-entity scope (D-03 corrected post-research: `Car` + `Track` added, `FeatureSettings` dropped, `PlayoffRound` reconciled).
- `BackupObjectMapperConfig` declares BOTH `@Primary` default AND `@Qualifier("backupObjectMapper")` strict mappers to preserve admin REST/AJAX behaviour while keeping backup serialization strict (Phase 72 D-11 amended post-research).
- `DataImportAudit` is a Lombok `@Entity` (NOT a record) because Hibernate 7 cannot proxy Java records; entity deliberately does NOT extend `BaseEntity` so Phase 75 writer fully controls `executedAt` without `AuditingEntityListener` interference (Phase 72 RESEARCH P-1).
- V7 columns use `LONGTEXT` (portable across H2 2.x and MariaDB 10.7+) for JSON-shape fields `table_counts_wiped` / `table_counts_restored` — native `JSON` rejected for v1.10 dialect drift (Phase 72 D-09).
- Replace-All implementation locked to native SQL `DELETE` in FK-reverse order + `em.flush() + em.clear()` + `JdbcTemplate.batchUpdate` for restore (bypasses `AuditingEntityListener`); TRUNCATE forbidden (auto-commits on MariaDB) — Phase 75.
- File-system mutations are NOT inside the JPA transaction; upload-tree restore is post-commit via stage-and-rename with `data/.import-backups/<ts>/uploads-old/` retained 24 h — Phase 75.
- Schema-version-strict-equality reigns for MVP; no forward-/backward-compat migration between schema versions (deferred to a future Migration Helper feature).
- Dockerfile pin `-noble` is a whitelist-on-suffix CI guard (`dockerfile-noble-pin-guard` job) mirroring the PLAT-07 `exec-maven-plugin` grep-gate from Phase 71; rationale comments inline in `Dockerfile` (Phase 78 D-04).
