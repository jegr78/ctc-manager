## Milestone Summary

Guest drivers can be assigned to a race lineup from **any** driver in the system, fielded for a chosen team; their results count toward the fielding team's score **and** credit the driver personally in the season's driver-ranking (including drivers with no `SeasonDriver` row that season); and guests are visibly marked on every surface that renders driver names. A final comments-only phase restored English-only / No-Comment-Pollution compliance after German comments crept in via v1.14/v1.15 Claude-Design handoffs.

## Key Accomplishments

- **Guest data model (113):** Flyway V18 additive `is_guest` column (V1–V17 untouched, H2 + MariaDB), `RaceLineup` guest flag as the single source of truth, full backup export/restore round-trip; admin can field any driver as a guest for any team, with edit/remove and result cascade.
- **Dual crediting without double-counting (114):** one attribution helper (home-team first, fielding-team fallback) feeds every driver-ranking path — including pure guests with no `SeasonDriver` row — while team scores flow through the existing `ScoringService.aggregateMatchScores`; idempotent on every result save (no template/controller fallback).
- **Consistent guest marking (115):** a single `--guest` CSS token + `hasGuestAppearance` flag drive the guest star/label across three Carbon-HUD graphics, admin race/matchday detail, admin + public driver-rankings, and the public driver-profile; visual treatment user-approved against a rendered reference.
- **Comment-language compliance (116):** every German comment translated to concise English or removed across templates, `application.yml`, `admin.css`, `logback-spring.xml`, one `src/main/java` Javadoc, and 10 test files; "Saison"→"Season" prose to match the `Season 2023` fixture; banned Phase/Plan markers stripped from touched files; zero behavioral change.

## Decisions

- **RaceLineup is the source of truth** for guest assignment (CLAUDE.md principle reinforced).
- **No separate points model** for guests — crediting reuses existing aggregation.
- **116 verification is the scan + green build, not new unit tests** (comments-only); the zero-German-comment check accepted as manual-only by signed-off decision (a permanent build-guard was offered and declined).
