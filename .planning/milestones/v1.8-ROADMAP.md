## Phase Details

### Phase 54: Preview Service & Row Categorization

**Goal:** A backend service exists that, given a Google Sheet URL, returns a structured preview categorizing every relevant row into one of six buckets, with no DB writes.

**Depends on:** Phase 53 (v1.6 milestone complete)
**Requirements:** IMPORT-02, IMPORT-03, IMPORT-04, IMPORT-05, UX-01, UX-02, UX-03, UX-04, UX-05, UX-06, MATCH-01, MATCH-02, DATA-01, DATA-02, DATA-04, DATA-05, TEST-01

**Success Criteria:**

1. `DriverSheetImportService.preview(sheetUrl)` returns a `DriverSheetImportPreview` containing exactly one `TabPreview` per sheet tab whose name matches `^\d{4}$`, sorted ascending by year, and ignores all non-matching tabs.
2. For each tab, every non-header row from columns A (PSN ID) and C (Team short code) is categorized into exactly one of: `NEW_DRIVER`, `NEW_ASSIGNMENT`, `CONFLICT`, `FUZZY_SUGGESTION`, `UNCHANGED`, `ERROR`.
3. Each `TabPreview` carries a `suggestedSeasonId` resolved via `SeasonRepository.findByYear(int)` (D-13 override of the original `findByName/findByDisplayLabel` wording — `Season.name` is free-text and `displayLabel` is computed, neither can back a JPA derived query).
4. Driver matching delegates to the existing `DriverMatchingService` 4-stage logic without modification.
5. Rows with blank PSN ID or unknown team short code are categorized as `ERROR`; no auto-create of `Season` or `Team` is attempted.
6. `DriverSheetImportServiceTest` covers preview categorization with at least 9 given-when-then scenarios; `./mvnw verify` passes.

**Plans:**

- [x] 54-01-PLAN.md — Preview service vertical slice: SeasonRepository.findByYear(int) + DriverSheetImportService (7 inner records + ErrorReason enum + D-12 waterfall) + DriverSheetImportServiceTest (16 @Test methods, 98.9% line coverage)

### Phase 55: Admin Import UI & Transactional Execute

**Goal:** An admin can click a button on `/admin/drivers`, submit a Sheet URL, review the per-tab preview with override controls, and execute the import transactionally with a flash summary.

**Depends on:** Phase 54
**Requirements:** IMPORT-01, IMPORT-06, UX-07, UX-08, DATA-03, TEST-02, TEST-03, QUAL-01, QUAL-02, QUAL-03, QUAL-04

**Success Criteria:**

1. Admin navigating to `/admin/drivers` sees an "Import from Google Sheet" button (CSS classes only, no inline styles) linking to `/admin/drivers/import`; submitting the Sheet URL form renders a per-tab preview page with one section per year-tab, each showing a pre-selected Season dropdown and six categorized row buckets with counts.
2. On the preview page, every `CONFLICT` row has a `Skip` checkbox (unchecked = overwrite with sheet value, checked = retain existing `SeasonDriver`), and every `FUZZY_SUGGESTION` row has an `Accept` checkbox (unchecked = treat as new driver, checked = link to suggested existing driver).
3. Clicking Execute performs all Driver creations and `SeasonDriver` upserts inside a single `@Transactional` boundary, then redirects to `/admin/drivers` with a flash summary; `RaceLineup` records remain untouched.
4. The controller contains no business logic, no repository calls, and no Google Sheets I/O — it delegates all work to `DriverSheetImportService`. **D-15 override:** form binding uses `@RequestParam` primitives + `Map<String, String>` (not the originally-worded `DriverSheetImportForm` DTO) because per-row keys (`seasonId_<year>`, `skip_<psnId>_<year>`, `accept_<psnId>_<year>`) are dynamic and don't fit a static DTO. Preview-state via re-fetch + form-params (D-06), no `@SessionAttributes`.
5. `DriverSheetImportControllerTest` (18 happy-path tests with mocked `GoogleSheetsService`) and `DriverSheetImportControllerExceptionTest` (4 exception-path tests) exercise the full `GET → POST /preview → POST /execute` flow; `./mvnw verify` passes the JaCoCo 82% line-coverage gate.

**Plans:**

- [x] 55-02-PLAN.md — Controller + templates + entry button: `DriverSheetImportController` (3 handlers) + `driver-import.html` + `driver-import-preview.html` (6 bucket tables, Skip/Accept controls) + drivers.html entry button
- [x] 55-03-PLAN.md — Integration tests: `DriverSheetImportControllerTest` (18 tests including 1 cross-tab fuzzy regression) + `DriverSheetImportControllerExceptionTest` (4 tests) + JaCoCo 82% gate green

### Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Reuse `GoogleSheetsService` + `DriverMatchingService` + `CsvImportController` preview-state pattern | No parallel import infrastructure | ✓ |
| `findByYear(int)` instead of `findByName/findByDisplayLabel` (D-13) | `Season.name` is free-text, `displayLabel` is computed; only `findByYear` is a viable JPA derived query | ✓ |
| `@RequestParam` + `Map<String,String>` instead of static Form DTO (D-15) | Per-row keys are dynamic — DTO would not fit | ✓ |
| Form-params re-fetch instead of `@SessionAttributes` (D-06) | Stateless controller, predictable transactional boundary | ✓ |
| Per-tab cache key for FUZZY-accept driver resolution (CR-01 fix) | Per-tab user choices must stay isolated; cross-tab dedup keeps plain-PSN key only for the no-accept branch | ✓ |
| Test years 2021/2022 in `DriverSheetImportControllerTest` | DevDataSeeder seeds 2023/2024/2026 on context startup → `findByYear()` ambiguity broke conflict-overwrite assertions | ✓ |
| ExecuteResult as mutable inner class (not record) | Matches `CsvImportService.ImportResult` precedent; simpler accumulation during the tab walk | ✓ |
| IOException → IllegalStateException for transactional rollback | Checked `IOException` does not trigger Spring `@Transactional` rollback by default | ✓ |

### Issues Encountered & Resolved

- **Stuck-agent recovery (Phase 55 Plan 03):** First executor agent hit a stream idle timeout after ~35 minutes with 39 tool uses. Orchestrator inspected the worktree, committed in-progress test class, completed remaining work inline (ExceptionTest authoring + year-fixture fix + `./mvnw verify`).
- **Year-fixture collision with DevDataSeeder:** Initial test run had 1 failure (`givenConflictRowWithoutSkip_whenExecute_thenSeasonDriverTeamOverwritten`) because `DevDataSeeder` seeds Seasons for 2023/2024/2026 on `@SpringBootTest(profiles=dev)` startup. Resolved by switching test fixtures to years 2021/2022.
- **Code review CR-01 (per-tab cache):** `crossTabCreatedDrivers` cache ignored per-tab `accept_<psnId>_<year>` overrides — first tab's choice silently won for subsequent tabs. Fixed via tab-scoped cache key for the FUZZY/accept path.
- **Code review WR-01 (exception leakage):** Exception messages from `IOException`/`DataAccessException`/`IllegalStateException` were leaking into flash attributes. Replaced with generic user-facing strings; full details remain in server-side logs.
- **Code review WR-02 (read-only transaction):** Added `@Transactional(readOnly = true)` to `preview()`.
- **Plan 02 → Plan 03 carryover fix:** `driver-import-preview.html` template had a SpEL `anyMatch` lambda inside `th:if` (CLAUDE.md "Keep Thymeleaf Templates Lean" violation + Thymeleaf-restricted-SpEL render failure). Hoisted to controller as `hasAmbiguousTabs` model attribute.
