From the new package `org.ctc.dataimport.exception` (NEW — Wave 1 creates):
```java
package org.ctc.dataimport.exception;

import java.io.IOException;

public abstract sealed class GoogleApiException extends IOException
    permits TransientGoogleApiException,
            AuthGoogleApiException,
            NotFoundGoogleApiException,
            PermissionGoogleApiException {

    public enum Category { TRANSIENT, AUTH, NOT_FOUND, PERMISSION }

    protected GoogleApiException(String message, Throwable cause) {
        super(message, cause);
    }

    public abstract Category category();
}
```

Each subtype is `public final class XGoogleApiException extends GoogleApiException` with `@Override public Category category() { return Category.X; }` and a single `(String message, Throwable cause)` constructor calling `super(message, cause)`. No Lombok; no Spring annotations.

Static mapper signature:
```java
public final class GoogleApiExceptionMapper {
    private GoogleApiExceptionMapper() {}
    public static GoogleApiException from(java.io.IOException e);
    public static AuthGoogleApiException from(java.security.GeneralSecurityException e);
}
```

Mapping table (from 91-RESEARCH.md § Pattern 2):

| Input | HTTP status / cause | Returned subtype |
|-------|---------------------|------------------|
| `GoogleJsonResponseException` | 401 (any reason) | `AuthGoogleApiException` |
| `GoogleJsonResponseException` | 403 + reason ∈ {authError, invalidCredentials, unauthorized} | `AuthGoogleApiException` |
| `GoogleJsonResponseException` | 403 + reason ∈ {forbidden, insufficientPermissions, other, null} | `PermissionGoogleApiException` |
| `GoogleJsonResponseException` | 404 (any reason) | `NotFoundGoogleApiException` |
| `GoogleJsonResponseException` | 408, 429, 500, 502, 503, 504 | `TransientGoogleApiException` |
| `GoogleJsonResponseException` | other / unknown | `TransientGoogleApiException` (lenient default) |
| `IOException` (non-Google: network/socket/timeout) | n/a | `TransientGoogleApiException` |
| `GeneralSecurityException` | n/a | `AuthGoogleApiException` (credentials unreadable) |

Defensive (Pitfall 11-like): mapper handles `gjre.getDetails() == null` and `errors().isEmpty()` cases by defaulting to `PermissionGoogleApiException` for 403 / `TransientGoogleApiException` otherwise.

Hardcoded user-visible message strings (D-09 — the `google-integration.md` Error Categories table is the SINGLE SOURCE OF TRUTH; controller catches MUST mirror exactly):

| Category | Hardcoded `errorMessage` literal |
|----------|----------------------------------|
| `Category.TRANSIENT` | `"Connection problem — retry"` |
| `Category.AUTH` | `"Authentication problem — re-link Google account"` |
| `Category.NOT_FOUND` | `"Sheet not found — check ID"` |
| `Category.PERMISSION` | `"Access denied — share the sheet with the service account"` |

NEVER include `e.getMessage()` in the user-visible flash text — per `<threat_model>` Threat T-91-02-IL info-leak mitigation. Log `e` at ERROR (Auth/Permission/NotFound) or WARN (Transient) WITH the cause chain, but the flash string is the hardcoded literal.

`errorCategory` flash key carries the `Category.name()` String — one of "TRANSIENT", "AUTH", "NOT_FOUND", "PERMISSION". Thymeleaf renders it via `th:classappend="|error-badge--${#strings.toLowerCase(errorCategory)}|"` to produce CSS class `error-badge--transient` etc.
