API contracts (per RESEARCH.md §Testcontainers Reuse Validation):
- `.withReuse(true)` is inherited from `GenericContainer<>` since 1.10.x → `MariaDBContainer<>` extends `JdbcDatabaseContainer<>` extends `GenericContainer<>`.
- The opt-in is per-developer-machine: file `~/.testcontainers.properties` containing the single line `testcontainers.reuse.enable=true`.
- Without the file, `.withReuse(true)` is a SILENT NO-OP — container cold-starts and Ryuk reaps at JVM exit (identical to today's behavior).
- With reuse enabled, Ryuk is SKIPPED — containers survive JVM exit and must be reaped via `docker container prune`.
- Container identity uses hash of `CreateContainerCmd` minus the random `sessionId` label; constant string literals on the chain keep the hash stable across runs (Testcontainers issue #2515 mitigation already built in — current declarations use only constant literals).

CI invariant (per Phase 77 D-05):
- Both ITs are gated by `@EnabledIfSystemProperty(named = "docker.available", matches = "true")`.
- CI runs `./mvnw verify -Pe2e` WITHOUT setting `-Ddocker.available=true`; both ITs are SKIPPED on CI.
- CI runners do not have `~/.testcontainers.properties`; even if the IT ran on CI, `.withReuse(true)` would be a no-op.
- Net CI impact: 0.
