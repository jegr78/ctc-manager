## Page Layout: driver-merge.html (Two States)

### State 1 — Select Target (GET /admin/drivers/{id}/merge)

**Primary visual anchor:** Target driver dropdown (`select` element) — the only interactive element requiring user input.

```
[toolbar]
  [back-link]  ← Back to Driver: {psnId}
  [h1]  Merge Driver: {psnId}
  [actions div — empty, no extra buttons]

[card]
  [detail-fields]
    SOURCE DRIVER (label)    {psnId} ({nickname})

  [form POST /admin/drivers/{id}/merge/preview]
    [form-group]
      [label]  Merge into (target driver)
      [select name="targetId" required]
        <option value="">-- Select target driver --</option>
        <option th:each ...>  {psnId} ({nickname})
    [actions]
      [btn btn-primary type=submit]  Select Target
      [a href=/admin/drivers/{id} btn btn-secondary]  Back to Driver
```

### State 2 — Preview + Confirm (GET /admin/drivers/{id}/merge?targetId={uuid} or POST /admin/drivers/{id}/merge/preview)

**Primary visual anchor:** "Confirm Merge" button (`btn-danger`) — the irreversible action requiring explicit user commitment.

```
[toolbar]
  [back-link]  ← Back to Driver: {psnId}
  [h1]  Merge Preview: {sourcePsnId} → {targetPsnId}
  [actions — empty]

[card]
  [h2]  References to Reassign

  [table-scroll]
    [table]
      [thead]  FK Table | Reassign | Duplicates Dropped
      [tbody]
        SeasonDriver     | {count}  | {count}
        RaceLineup       | {count}  | {count}
        RaceResult       | {count}  | {count}
        PsnAlias         | {count}  | —

  [form POST /admin/drivers/{id}/merge]
    [input type=hidden name=targetId value={targetId}]
    [actions]
      [btn btn-danger onclick=confirm(...)]  Confirm Merge
      [a href=/admin/drivers/{id}/merge btn btn-secondary]  Back
```

Source: CONTEXT.md D-07, D-08, D-10, D-14.

## Interaction Contract

### Merge Button on driver-detail.html (D-18)

Position: In `.actions` div, between Edit and Delete buttons.

```html
<a th:href="@{/admin/drivers/{id}/merge(id=${driver.id})}" class="btn btn-secondary">Merge</a>
```

No JavaScript required on this button — it is a plain link.

### Confirm Merge Button (D-11)

JavaScript `confirm()` guard, consistent with delete pattern:

```javascript
onclick="return confirm('Really merge {sourcePsnId} into {targetPsnId}? This cannot be undone.')"
```

The confirm text includes both driver PSN IDs (rendered server-side via `th:attr="onclick=..."` or inline Thymeleaf).

### Form Flow

1. GET `/admin/drivers/{id}/merge` — renders step 1 (select target)
2. POST `/admin/drivers/{id}/merge/preview` — renders step 2 (preview counts)
3. POST `/admin/drivers/{id}/merge` — executes merge, redirects to target driver detail

No AJAX, no modals — pure server-side form submits (D-03).

### Redirect After Success (D-12)

`RedirectAttributes.addFlashAttribute("successMessage", "Driver merged: {sourcePsnId} into {targetPsnId} — {total} references reassigned, {dropped} duplicates resolved")`

Rendered by existing `th:if="${successMessage}"` flash-message block in layout.html.

### Error Handling (D-13)

On exception (source already deleted, target not found):
`RedirectAttributes.addFlashAttribute("errorMessage", "Merge failed: {reason}. Return to the driver list and verify both drivers exist.")`

Redirect to `/admin/drivers`.
