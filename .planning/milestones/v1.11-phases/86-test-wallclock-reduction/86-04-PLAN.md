<objective>
Convert the 3 domain Phase repository ITs from `@SpringBootTest + @Transactional` to `@DataJpaTest + @Import(JpaAuditingConfig.class)`. Demonstrate the slice pattern is viable for future conversions while preserving assertion parity. Inline-seed `RaceScoring` and `MatchScoring` in `@BeforeEach` to replace the Flyway-dependent `findAll().get(0)` calls.

Purpose: PERF-03 mandates at least one repository IT converted from `@SpringBootTest` to `@DataJpaTest` slice. D-05/D-06 widen this to all 3 Phase repository ITs to establish the pattern. The slice loads only JPA infrastructure (no controllers, services, security, scheduling) and starts substantially faster than full `@SpringBootTest`. The `JpaAuditingConfig` is required because `@DataJpaTest` does not load `CtcManagerApplication` (which carries `@EnableJpaAuditing`).

Output: New `JpaAuditingConfig` `@TestConfiguration` in test classpath, 3 converted ITs each with `@BeforeEach` scoring fixtures, assertion-parity confirmed via `./mvnw verify -Dit.test=` per IT, and a note in 86-04-SUMMARY about the per-IT context-load count contribution.
</objective>
