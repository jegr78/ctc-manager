  <what-built>
    Plans 01-05 are complete on the milestone PR branch (`gsd/v1.11-tooling-and-cleanup`). Per D-17, CI runs against the PR branch are semantically equivalent to post-merge master runs because `ci.yml` executes identical steps regardless of trigger event. The `workflow_dispatch` trigger has been added to ci.yml in this phase so that 5 consecutive runs can be collected sequentially without the natural-push path being blocked by `concurrency.cancel-in-progress: true`. Operator triggers 5 runs sequentially, waiting for each to complete before triggering the next, then harvests Maven `Total time` from each run.
  </what-built>
  <how-to-verify>
    Step 1 — confirm `workflow_dispatch` is wired and the PR branch is current:

    ```bash
    grep -q "workflow_dispatch:" .github/workflows/ci.yml && echo "PASS: workflow_dispatch present"
    gh pr view 122 --json state,mergeable,headRefName --jq '"state=\(.state) mergeable=\(.mergeable) head=\(.headRefName)"'
    ```

    Step 2 — sequentially trigger 5 ci.yml runs on the milestone PR branch, waiting for each:

    ```bash
    BRANCH=gsd/v1.11-tooling-and-cleanup
    for i in 1 2 3 4 5; do
      echo "=== Triggering run $i/5 ==="
      gh workflow run ci.yml --ref "$BRANCH"
      sleep 5  # let GitHub register the run
      RUN_ID=$(gh run list --workflow=ci.yml --branch="$BRANCH" --event=workflow_dispatch --limit=1 --json databaseId --jq '.[0].databaseId')
      echo "Watching run $RUN_ID..."
      gh run watch "$RUN_ID" --exit-status
    done
    ```

    Sequential waiting is mandatory — `concurrency.cancel-in-progress: true` would kill any earlier run if a new one starts before the previous completes (D-17).

    Step 3 — extract Maven Total time from each of the 5 workflow_dispatch runs:

    ```bash
    BRANCH=gsd/v1.11-tooling-and-cleanup
    for id in $(gh run list --workflow=ci.yml --branch="$BRANCH" --event=workflow_dispatch --status=success --limit=5 --json databaseId --jq '.[].databaseId'); do
      echo "=== Run $id ==="
      gh run view "$id" --log 2>&1 | grep -E "BUILD SUCCESS|\[INFO\] Total time:" | tail -5
    done
    ```

    Record the 5 Maven "Total time" values. Drop min and max. Compute median of the 3 middle values.

    Step 4 — variance check (D-10): if (max − min) / median > 20%, repeat Step 2 with a second 5-run block.
  </how-to-verify>
