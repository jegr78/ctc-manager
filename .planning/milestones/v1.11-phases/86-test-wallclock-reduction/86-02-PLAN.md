<objective>
Apply the D-04 sitegen cluster root-cause fix: replace the shared-singleton `SiteProperties.outputDir` mutation pattern with per-class `@TempDir` + `@DynamicPropertySource` binding so each sitegen test class owns its own output directory before Spring builds the context. After the cluster fix lands, conditionally remove `TestDataServiceIntegrationTest`'s defensive `@DirtiesContext` (Cluster C) if random-seed verification confirms the upstream pollution is gone.

Purpose: 7 of the 11 `@DirtiesContext` annotations in the suite (RESEARCH Cluster A) share one root cause — `siteProperties.setOutputDir(tempDir.toString())` mutating a singleton bean in `@BeforeAll`. Removing them as a cluster is the largest single context-eviction reduction in the phase. RESEARCH Open Question 2 flags the tradeoff: distinct `@TempDir` per class may produce 7 separate context startups instead of 1 shared-with-evictions; the ContextLoadCountListener (Plan 01) quantifies whether this is a net win, but the structural fix is correct independent of the count.

Output: 7 modified sitegen test files (no longer mutate SiteProperties), 1 conditionally modified TestDataServiceIntegrationTest, 3-seed verification evidence for each removal, and an entry in docs/test-performance.md Per-Decision Evidence section appended by the Wave 3 summary.
</objective>
