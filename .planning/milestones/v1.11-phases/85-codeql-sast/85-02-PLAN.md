## Phase-81-D-10 Decision Tree (templated for D-15 live triage)

For every HIGH or CRITICAL baseline alert:

1. **HIGH/CRITICAL real bug** → fix in source, commit `fix(85): triage <rule-id> on <ClassName>`, set bucket `fixed` in sast-acceptance.md with commit SHA in `Source-Marker` column. NO `// CodeQL FP:` marker (finding is closed, not suppressed).
2. **HIGH/CRITICAL intentional pattern (defense-in-depth via Phase 81 / unit-tested / sanitizer-not-recognized)** → suppress via `codeql-config.yml` `query-filters` exclude + `// CodeQL FP:` source marker + `sast-acceptance.md` row with bucket `suppressed`. Commit `chore(85): suppress <rule-id> on <ClassName> (FP rationale)`.
3. **HIGH/CRITICAL stylistic / low-actionability** → same as #2 with shorter rationale; commit `chore(85): suppress <rule-id> on <ClassName> (low-actionability)`.
4. **Cannot triage (insufficient evidence, ambiguous semantics)** → escalate: leave the alert open, document in `85-VERIFICATION.md` with bucket `accepted` (operator-decision), commit `docs(85): accept <rule-id> on <ClassName> (escalated risk)`. RARE — should only fire when the alert genuinely defies classification.
