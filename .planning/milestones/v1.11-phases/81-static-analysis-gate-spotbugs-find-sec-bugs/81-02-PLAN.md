<objective>
Run the baseline SpotBugs report from PLAN 01's plumbing commit, walk every
Medium+HIGH BugInstance through the D-10 decision tree (HIGH must fix; Medium
real bug must fix; Medium intentional pattern suppress with rationale + code
cross-ref; Medium stylistic suppress with short reason), and execute N atomic
triage commits — fixes use `fix(81): ...`, new suppressions use `chore(81): ...`
— until `./mvnw spotbugs:spotbugs -DskipTests` reports ZERO Medium+HIGH
BugInstances.

Purpose: STAT-05 interleaved triage between the two STAT-05 atomic commits per
D-12 step 3. This is the gate-readiness work — once finished, PLAN 03's
gate-flip commit will succeed cleanly on `./mvnw verify` and `./mvnw verify -Pe2e`.

Output:
- Populated `## STAT-05 Triage Table` section in `81-VERIFICATION.md` listing
  every Medium+HIGH BugInstance with disposition + rationale.
- N atomic git commits on the same feature branch as PLAN 01 (one per
  pattern-family or per file), each scoped per D-12 step 3.
- Augmented `config/spotbugs-exclude.xml` with all triage-time `<Match>`
  entries (every one carrying a rationale comment per D-09).
- Source code changes: (a) real-bug fixes for HIGH + Medium-real findings,
  (b) inline cross-reference comments in `FileStorageService.java` +
  `BackupArchiveService.java` (per PATTERNS.md sections 5 + 6) so a reviewer
  reading the source can find the suppression rationale without opening the
  filter file.
- Baseline state: `./mvnw spotbugs:spotbugs -DskipTests` reports zero
  Medium+HIGH BugInstance after the final triage commit. JaCoCo coverage
  remains >= 82%.

Why this plan is NOT autonomous: D-10 fix-vs-suppress is a human judgement call
per finding. The Manual-Only rows in `81-VALIDATION.md` (Baseline-report triage
decisions; pom.xml plugin block visual sanity) require executor + user
collaboration on the triage table before the chore/fix commits are made. The
autonomous flag is false until the user approves the populated triage table.
