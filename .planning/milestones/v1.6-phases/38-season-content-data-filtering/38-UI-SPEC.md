## Component Inventory

### New CSS class: `.season-meta`

Used beneath section titles and in the hero to display year and season number context.

```
.season-meta {
    font-size: 12px;
    color: var(--text-muted);
    text-transform: uppercase;
    letter-spacing: 1px;
    margin-top: 4px;
}
```

Placement: immediately after the parent heading element (`.section-title` or `.hero h1`).

**Rendered format:** `{year} | Season #{number}` — e.g., `2025 | Season #3`

### Modified: `.hero-label` (index.html)

Current: `Season {season.name}` (plain text label above `<h1>`).
New: `Season {season.name} — {season.year}` — year appended to hero-label in the
same element. No new element required; the hero-label absorbs the year.

Rationale: The hero-label already uses `.hero-label` at 11px / muted / uppercase —
no visual conflict with the `<h1>` below.

### Modified: `.match-meta` (matchday.html, index.html)

The entire `.match-meta` div is conditionally rendered. When both `race.track` and
`race.car` are null, the element must not appear in the DOM (not hidden via CSS — it
must not exist).

Guard expression: `th:if="${race.track != null or race.car != null}"`

When only one field is present, no separator dash is rendered. The empty-string
fallback (`?: ''`) already prevents null output; the separator `<span>` has its own
`th:if` guard that already exists and is correct.

### Modified: archive.html — Season column

The Season column cell shows name plus year and number in a secondary line:
```
{season.name}          ← font-bold text-white (existing)
{year} | #{number}     ← new .season-meta class, same cell, block-level
```

Implementation: two elements inside the `<td>`, second element uses `.season-meta`.

### Modified: archive.html — Period column

When both `startDate` and `endDate` are null, the cell renders nothing (empty `<td>`).
When only one date is present, only that date renders — no orphaned ` — ` separator.
Current guards are almost correct; tighten the separator condition (already `startDate
!= null and endDate != null`). No visual change needed — the existing `.text-dim`
styling applies.

### No new page or navigation element

This phase adds NO new pages, routes, or navigation items. All changes are
in-place enrichment of existing templates.

## Interaction Contract

### Conditional visibility rules

| Element | Condition | Behavior |
|---------|-----------|----------|
| `.match-meta` div | `race.track == null AND race.car == null` | Not rendered (DOM absent) |
| Period separator ` — ` | `startDate == null OR endDate == null` | Not rendered |
| Period cell content | Both dates null | Empty `<td>`, no text |
| Season archive row | `season.name.contains("Test")` | Row not generated (filtered at service level, not template level) |

### No JavaScript interaction

All conditional rendering uses Thymeleaf `th:if` / `th:unless` server-side. No
client-side toggling, no JS changes in this phase.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Season metadata subtitle | `{year} \| Season #{number}` (e.g. `2025 \| Season #3`) |
| Hero label (enriched) | `Season {season.name} — {season.year}` |
| Archive season column, secondary line | `{year} \| #{number}` |
| Team profile season context | `Season {season.name}` (unchanged — no year enrichment here per scope) |
| Driver profile season context | `(Season {season.name})` inline (unchanged) |
| Empty match-meta | (element absent — no placeholder copy) |
| Empty period cell | (empty cell — no em-dash, no placeholder copy) |
| Primary CTA | "Standings" (archive table link — existing, unchanged) |
| Empty state heading | n/a — no data-empty-state page in this phase |
| Error state | n/a — no error UI in this phase |
| Destructive confirmation | n/a — no destructive actions in this phase |

**Copywriting notes:**
- Pipe character `|` is the established separator from `Season.getDisplayLabel()`;
  use consistently.
- Season number formatted as `#3` (hash prefix), not `No. 3` or `3`.
- Year is a four-digit integer, no formatting needed.
- "Test" seasons are fully absent from rendered output — no "filtered" message shown.
