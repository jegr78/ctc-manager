With the iFrame Player API block:
```html
<!-- YouTube Hero Video — iFrame Player API for reliable autoplay + loop (Phase 51: YT-01, YT-02) -->
<div class="landing-hero" th:if="${videoId != null and !#strings.isEmpty(videoId)}">
    <div id="yt-hero-player"></div>
    <div class="landing-hero-overlay"></div>
</div>
<script th:if="${videoId != null and !#strings.isEmpty(videoId)}" th:inline="javascript">
(function() {
    var videoId = /*[[${videoId}]]*/ '';
    var tag = document.createElement('script');
    tag.src = 'https://www.youtube.com/iframe_api';
    document.head.appendChild(tag);

    window.onYouTubeIframeAPIReady = function() {
        new YT.Player('yt-hero-player', {
            videoId: videoId,
            playerVars: {
                autoplay: 1,
                mute: 1,
                controls: 0,
                loop: 1,
                playlist: videoId,
                rel: 0,
                playsinline: 1,
                disablekb: 1,
                fs: 0,
                iv_load_policy: 3
            },
            events: {
                onReady: function(event) {
                    event.target.playVideo();
                },
                onStateChange: function(event) {
                    if (event.data === YT.PlayerState.ENDED) {
                        event.target.seekTo(0);
                        event.target.playVideo();
                    }
                }
            }
        });
    };
})();
</script>
```

Key parameter details:
- `th:inline="javascript"` enables Thymeleaf's JavaScript inlining mode
- `/*[[${videoId}]]*/` is Thymeleaf's safe inline expression (auto-escapes, falls back to empty string for static prototype)
- The IIFE prevents global scope pollution
- `autoplay:1 + mute:1` is mandatory (browsers block autoplay with sound)
- `loop:1 + playlist:videoId` is native loop as first defense
- `onStateChange ENDED -> seekTo(0) + playVideo()` is the safety net for seamless looping
- `controls:0, disablekb:1, fs:0` hides all player controls
- `iv_load_policy:3` suppresses annotations
- `playsinline:1` prevents iOS fullscreen takeover
- `rel:0` prevents related videos from other channels
