## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, nav link gaps |
| md | 16px | Table cell padding (10px vertical, 16px horizontal), default margin |
| lg | 24px | Team header gap, section element spacing |
| xl | 32px | Main content padding, section margin-bottom |
| 2xl | 48px | Hero padding, footer top margin |
| 3xl | 64px | Bracket round gap (largest spacing tier) |

Exceptions: Nav height is 60px (not a spacing token — fixed component dimension). Touch target for `.entity-link` must be at minimum 44px tall in mobile context; achieve via `padding: 8px 0` on `<a>` within list items.

**Source:** Measured from existing `style.css` values. No tailwind — values are inline CSS.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.6 (from `body` rule) |
| Label / table header | 11px | 400 (regular) | 1.4 |
| Section title | 13px | 600 (Conthrax semibold) | 1.2 |
| Display / hero heading | 28px (desktop), 22px (mobile) | 600 (Conthrax semibold) | 1.2 |

**Rules for entity links specifically:**
- Driver PSN-ID links: 14px, weight 700 (bold — inherits `.font-bold` from `<td>` context), color `var(--accent)`
- Team short-name links in standings: 14px, weight 700 (inherits `<strong>` context), color `var(--accent)`
- Driver list on team profile: 14px, weight 400, color `var(--accent)`

**Source:** `style.css` `table { font-size: 14px }`, `th { font-size: 11px }`, `section-title { font-size: 13px }`, `hero h1 { font-size: 28px }`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (`--bg`) | Page background, all surfaces behind content |
| Secondary (30%) | `#111` (`--bg-card`) | Cards, nav bar, table wrappers, match cards |
| Accent (10%) | `#4fc3f7` (`--accent`) | Entity cross-links (team names, driver names), match scores, bracket winners |
| Destructive | n/a | No destructive actions in this phase |

Accent reserved for:
1. `.entity-link` — clickable team names (standings table, index standings)
2. `.entity-link` — clickable driver PSN-IDs (driver ranking table, matchday results, driver list on team profile)
3. Match score display (`.match-score`) — already established
4. Bracket winner highlight (`.bracket-team.winner`) — already established

**Hover state for `.entity-link`:** `color: #b3e5fc` (lighter tint of accent, approximately 70% lighter) + `text-decoration: underline`. Transition: `all 0.2s` (consistent with existing `.nav-links a` transition).

**Source:** `style.css` `:root { --accent: #4fc3f7 }`. Decision D-05, D-06 from `39-CONTEXT.md`.

## Component Inventory

### New: `.entity-link` CSS class

Add to `style.css`. Full specification:

```css
.entity-link {
    color: var(--accent);
    text-decoration: none;
    transition: all 0.2s;
}

.entity-link:hover {
    color: #b3e5fc;
    text-decoration: underline;
}
```

Apply to: every `<a>` element that links a team name or driver name to a profile page.

### Modified: standings table team cell (`standings.html`, `index.html`)

Current:
```html
<strong class="text-white" th:text="${s.team.shortName}"></strong>
<span class="text-dim" th:text="' ' + ${s.team.name}"></span>
```

Target pattern:
```html
<a class="entity-link font-bold" th:href="${s.teamProfileUrl}" th:text="${s.team.shortName}"></a>
<span class="text-dim" th:text="' ' + ${s.team.name}"></span>
```

- `font-bold` inherited from existing class; `text-white` removed (replaced by accent color)
- `teamProfileUrl` is a pre-computed relative path from the service (Decision D-08, D-09)

### Modified: driver ranking table (`driver-ranking.html`)

Current:
```html
<td class="font-bold text-white" th:text="${r.driver.psnId}"></td>
```

Target pattern:
```html
<td class="font-bold">
    <a class="entity-link" th:href="${r.driverProfileUrl}" th:text="${r.driver.psnId}"></a>
</td>
```

- `text-white` removed from `<td>`, accent color comes from `.entity-link`
- `driverProfileUrl` pre-computed in service

### Modified: matchday result rows (`matchday.html`)

Current:
```html
<td class="font-bold" th:text="${result.driverPsnId}"></td>
```

Target pattern:
```html
<td class="font-bold">
    <a class="entity-link" th:href="${result.driverProfileUrl}" th:text="${result.driverPsnId}"></a>
</td>
```

- `driverProfileUrl` added to `RaceView.ResultView` record

### New: Driver listing section (`team-profile.html`)

Add below the Record table. Specification:

```html
<div class="section" th:if="${not #lists.isEmpty(drivers)}">
    <h2 class="section-title">Drivers</h2>
    <div class="table-wrap">
        <table>
            <thead>
                <tr><th>Driver</th><th class="text-right">Points</th></tr>
            </thead>
            <tbody>
                <tr th:each="d : ${drivers}">
                    <td>
                        <a class="entity-link font-bold" th:href="${d.driverProfileUrl}"
                           th:text="${d.psnId}"></a>
                    </td>
                    <td class="text-right font-bold text-white" th:text="${d.totalPoints}"></td>
                </tr>
            </tbody>
        </table>
    </div>
</div>
```

- Section is hidden when `drivers` list is empty (guard with `th:if`)
- `driverProfileUrl` is a pre-computed relative path from `generateTeamProfiles()`
- `totalPoints` is pre-computed in service (Decision D-03)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None — this phase adds hyperlinks, not buttons or forms |
| Driver section heading | `Drivers` (all-caps via `section-title` CSS class) |
| Empty driver list | Section hidden entirely — no empty state message (guard via `th:if="${not #lists.isEmpty(drivers)}"`) |
| Empty state (no standing on team profile) | Already handled by existing `th:if="${standing != null}"` |
| Error state | No new error paths introduced; link rendering is null-safe via pre-computed URLs |
| Destructive confirmation | None — no destructive actions in this phase |

**Attribute labels (not visible text, but HTML semantics):**
- `<a>` elements wrapping team/driver names carry no `title` attribute — the link text is self-describing
- No `aria-label` needed for entity links (link text equals entity name)

**Source:** Decisions from `39-CONTEXT.md` (D-01, D-03). Section heading follows existing `section-title` pattern (`standings.html`: "Team Standings", `team-profile.html`: "Record").

## Interaction States

| State | Spec |
|-------|------|
| Default | `color: var(--accent)` (`#4fc3f7`), `text-decoration: none` |
| Hover | `color: #b3e5fc`, `text-decoration: underline`, `transition: all 0.2s` |
| Focus (keyboard) | Browser default outline (`:focus-visible`) — do not suppress |
| Visited | No visited color override — keep accent color for visited links (design consistency) |
| Disabled / null | Render as plain text when URL is null — guard with `th:if="${url != null}"` / `th:else` fallback |
