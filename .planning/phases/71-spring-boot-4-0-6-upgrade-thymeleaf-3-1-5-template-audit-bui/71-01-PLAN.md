<objective>
Eliminate every `${...}`-expression sitting in a `th:replace/insert/include` fragment-call argument across the 10 affected admin templates and wire the value through a controller-supplied `pageTitle` model attribute. Add a null-safe Elvis fallback in `admin/layout.html` so a missing `pageTitle` cannot cause an NPE on the `${title}.contains(...)` sidebar-class logic. This plan runs on Spring Boot 4.0.5 (the platform bump lands in Plan 03) — refactoring the templates first means the SB 4.0.6 bump in Plan 03 has zero remaining offenders to break against.

Purpose: PLAT-03 + PLAT-04 (admin half) — every fragment-parameter expression flagged by PATTERNS.md becomes the controller-side `pageTitle` pattern recommended by the Thymeleaf maintainer in thymeleaf/thymeleaf#1082, aligning with CLAUDE.md "Keep Thymeleaf Templates Lean".

Output:
- 10 admin templates with `${pageTitle}` fragment-call argument (was: ternaries, Elvis, string-concat)
- 1 admin layout with Elvis-safe title rendering + null-safe sidebar classes
- 9 controller GET-handler additions across 7 admin controllers (one helper edit covers two SeasonPhaseController routes; one helper edit covers two SeasonPhaseGroupController routes; SeasonPhaseController.detail/groupDetail both render season-detail.html)
- `./mvnw verify` BUILD SUCCESS on the still-4.0.5 baseline with all 1227 unit + 31 Playwright E2E green
</objective>
