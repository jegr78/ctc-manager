**Part B — Add `<dependencyManagement>` block with Thymeleaf 3.1.5 pin:**

Insert a new `<dependencyManagement>` block immediately AFTER the closing `</properties>` line
(currently around line 20) and BEFORE the opening `<dependencies>` line (currently around line 21).
This is the conventional Maven placement.

Exact insertion:
```xml
	<dependencyManagement>
		<dependencies>
			<dependency>
				<groupId>org.thymeleaf</groupId>
				<artifactId>thymeleaf</artifactId>
				<version>3.1.5</version>
			</dependency>
		</dependencies>
	</dependencyManagement>
```

(Match the existing pom.xml indentation style: tabs for nesting.)

**Rationale (per CONTEXT.md D-16 + STACK.md L26-34):**
- Spring Boot 4.0.6 already ships Thymeleaf 3.1.5 transitively via its parent BOM. The
  `<dependencyManagement>` pin is forward-compat protection — preventing a future Spring Boot
  4.0.7 / 4.0.8 patch release from silently introducing a Thymeleaf 3.1.6+ with potentially
  stricter restricted-mode evaluation that could re-break already-fixed templates.
- No new Maven dependency is being introduced (D-17). `org.thymeleaf:thymeleaf` is already on
  the classpath transitively via `spring-boot-starter-thymeleaf`. The `<dependencyManagement>`
  block only pins the version Maven resolves for that existing artifact.
