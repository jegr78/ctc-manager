```java
@Autowired private RequestMappingHandlerMapping handlerMapping;

Stream<DynamicTest> allAdminGetRoutes() {
    return handlerMapping.getHandlerMethods().entrySet().stream()
        .filter(e -> e.getKey().getMethodsCondition().getMethods().contains(RequestMethod.GET)
                  || e.getKey().getMethodsCondition().getMethods().isEmpty())  // @RequestMapping defaults to all methods
        .flatMap(e -> e.getKey().getPathPatternsCondition() != null
                ? e.getKey().getPathPatternsCondition().getPatternValues().stream()
                : e.getKey().getPatternsCondition().getPatterns().stream())
        .filter(p -> p.startsWith("/admin"))
        .map(p -> DynamicTest.dynamicTest("GET " + p, () -> assertRouteRenders(p)));
}
```

The `getPathPatternsCondition()` vs `getPatternsCondition()` distinction depends on whether the
app uses `PathPatternParser` (Spring 6+ default) or the legacy `AntPathMatcher`. CTC is on Spring
7 — `getPathPatternsCondition()` is available. If both return null in a particular handler,
fall back to an empty stream.

```java
package org.ctc.admin.controller.integration;

import org.junit.jupiter.api.DynamicTest;
import org.junit.jupiter.api.TestFactory;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.context.SpringBootTest;
import org.springframework.boot.webmvc.test.autoconfigure.AutoConfigureMockMvc;
import org.springframework.test.context.ActiveProfiles;
import org.springframework.test.context.jdbc.Sql;
import org.springframework.test.web.servlet.MockMvc;
import org.springframework.transaction.annotation.Transactional;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.servlet.mvc.method.RequestMappingInfo;
import org.springframework.web.servlet.mvc.method.annotation.RequestMappingHandlerMapping;

import java.util.LinkedHashSet;
import java.util.Map;
import java.util.Set;
import java.util.stream.Stream;

import static org.assertj.core.api.Assertions.assertThat;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.get;

@SpringBootTest
@AutoConfigureMockMvc
@ActiveProfiles("dev")
@Transactional
@Sql(scripts = "/sql/template-rendering-smoke-fixture.sql",
        executionPhase = Sql.ExecutionPhase.BEFORE_TEST_METHOD)
class TemplateRenderingSmokeIT {
    // Path-variable substitution map (per CONTEXT.md D-10) — keys cover every path-var name
    // any /admin/** route declares; values are the seeded UUIDs from
    // src/test/resources/sql/template-rendering-smoke-fixture.sql.
    private static final Map<String, String> PATH_VARS = Map.ofEntries(
            Map.entry("id",                  "00000000-0000-0071-0000-000000000010"), // generic id — defaults to season (most common case)
            Map.entry("seasonId",            "00000000-0000-0071-0000-000000000010"),
            Map.entry("phaseId",             "00000000-0000-0071-0000-000000000011"),
            Map.entry("groupId",             "00000000-0000-0071-0000-000000000012"),
            Map.entry("teamId",              "00000000-0000-0071-0000-000000000020"),
            Map.entry("driverId",            "00000000-0000-0071-0000-000000000070"),
            Map.entry("matchdayId",          "00000000-0000-0071-0000-000000000050"),
            Map.entry("matchId",             "00000000-0000-0071-0000-000000000055"),
            Map.entry("raceId",              "00000000-0000-0071-0000-000000000060"),
            Map.entry("raceScoringId",       "00000000-0000-0071-0000-000000000001"),
            Map.entry("matchScoringId",      "00000000-0000-0071-0000-000000000002"),
            Map.entry("sourceId",            "00000000-0000-0071-0000-000000000070"),  // driver-merge source
            Map.entry("targetId",            "00000000-0000-0071-0000-000000000071"),  // driver-merge target
            Map.entry("playoffId",           "00000000-0000-0071-0000-000000000099")   // omit playoff in fixture; this entry exists to suppress NPE when the route is enumerated even if it 404s
    );

    @Autowired private MockMvc mockMvc;
    @Autowired private RequestMappingHandlerMapping handlerMapping;

    /**
     * Discovers every /admin/** GET route via RequestMappingHandlerMapping introspection
     * and asserts each (a) returns HTTP 200 or 3xx and (b) does not contain
     * "TemplateProcessingException" in the response body. Path variables are substituted
     * from a static map keyed on path-variable name to seeded fixture UUID.
     */
    @TestFactory
    Stream<DynamicTest> givenSmokeFixtureSeeded_whenGetAllAdminRoutes_thenAllRenderWithoutTemplateProcessingException() {
        Set<String> patterns = new LinkedHashSet<>();
        handlerMapping.getHandlerMethods().forEach((info, method) -> {
            if (!supportsGet(info)) return;
            Set<String> ps = info.getPathPatternsCondition() != null
                    ? info.getPathPatternsCondition().getPatternValues().stream()
                            .map(p -> p.getPatternString()).collect(java.util.stream.Collectors.toCollection(LinkedHashSet::new))
                    : (info.getPatternsCondition() != null
                            ? new LinkedHashSet<>(info.getPatternsCondition().getPatterns())
                            : java.util.Collections.emptySet());
            for (String p : ps) {
                if (p != null && p.startsWith("/admin")) {
                    patterns.add(p);
                }
            }
        });

        return patterns.stream().map(p -> DynamicTest.dynamicTest("GET " + p, () -> assertRouteRenders(p)));
    }

    private static boolean supportsGet(RequestMappingInfo info) {
        var methods = info.getMethodsCondition().getMethods();
        return methods.isEmpty() || methods.contains(RequestMethod.GET);
    }

    private void assertRouteRenders(String pattern) throws Exception {
        String url = substitutePathVars(pattern);
        var response = mockMvc.perform(get(url)).andReturn().getResponse();

        // Accept 200 OK, 3xx redirects (some admin routes intentionally redirect, e.g., season detail → phase detail),
        // or 404 ONLY if the path variable is not seeded (defensive — log but don't fail).
        // We DO fail on 5xx and on any response body that contains TemplateProcessingException.
        int status = response.getStatus();
        String body = response.getContentAsString();

        assertThat(status)
                .as("GET %s must not return 5xx (status was %d)", url, status)
                .isLessThan(500);

        assertThat(body)
                .as("GET %s response body must not contain TemplateProcessingException — "
                        + "this signals a Thymeleaf restricted-mode failure (PLAT-06)", url)
                .doesNotContain("TemplateProcessingException");
    }

    private String substitutePathVars(String pattern) {
        String result = pattern;
        for (Map.Entry<String, String> entry : PATH_VARS.entrySet()) {
            result = result.replace("{" + entry.getKey() + "}", entry.getValue());
        }
        // Strip any remaining {placeholder} segments (route declares a path var we did not seed):
        // replace with seeded season UUID as a best-effort default. The TemplateProcessingException
        // check still works against the response.
        result = result.replaceAll("\\{[^/}]+\\}", PATH_VARS.get("id"));
        return result;
    }
}
```

- The `PATH_VARS` map covers every path-variable name documented in PATTERNS.md § 5 controllers
  (`seasonId`, `phaseId`, `groupId`, `teamId`, `driverId`, `matchdayId`, `matchId`, `raceId`,
  `raceScoringId`, `matchScoringId`, `sourceId`, `targetId`) plus generic `id` defaulted to the
  seeded season UUID.
- If a route declares an unseeded path variable (e.g., a future `/admin/import/{importId}/`),
  the `replaceAll` fallback at the end of `substitutePathVars` substitutes the season UUID — the
  route will likely 404 but the TemplateProcessingException assertion still runs against the 404
  page body, which is exactly what we want (a 404 error page must NOT throw a TemplateProcessingException).

**Assertion contract (per D-11):**
- Status check: `< 500` (allows 200, 3xx redirects, 4xx if path-var isn't found — these are
  legitimate runtime outcomes, NOT template-rendering failures). The original D-11 says
  `status().isOk()`, but admin routes like `/admin/seasons/{id}` (which redirects to a phase
  URL) make a strict isOk() check fragile. The looser `< 500` check is the actual goal — a
  TemplateProcessingException would manifest as a 500 with the exception name in the body,
  which IS caught by the body-contains check.
- Body check: `doesNotContain("TemplateProcessingException")` — exact substring match. Word-
  boundary anchoring is unnecessary because the substring is unique to Thymeleaf's exception
  hierarchy; no domain-level usage of the term exists in CTC code.

**@Sql execution phase (per D-09):**
- `Sql.ExecutionPhase.BEFORE_TEST_METHOD` is the safe default — runs before EACH dynamic test.
- Combined with `@Transactional` at class level, each dynamic test executes inside its own
  transactional boundary that rolls back at the end. The fixture's `INSERT`s are visible to
  the controller-handler call but disappear before the next dynamic test starts; the next
  test re-runs `@Sql` to re-seed. No PK collision risk.

**Why NOT introspect path-variable-to-entity-type from controller method signatures:**
- Reflection on `@PathVariable` annotations would let us route `{id}` in `DriverController.detail`
  to the driver UUID, vs `{id}` in `RaceController.detail` to the race UUID. We deliberately
  skip this complexity (per D-08 / D-10): the generic `id` slot maps to season; any handler
  expecting a different entity type will return 404, the 404 page renders cleanly (no
  TemplateProcessingException), and the assertion passes. The smoke IT is about template
  rendering, NOT route correctness.
