In `src/main/resources/templates/admin/layout.html`, insert the following block inside the `<head>` section, between the existing `<link rel="stylesheet" ...>` tag (line 9) and the closing `</head>` tag (line 10):

```html
<th:block th:if="${_csrf != null}">
    <meta name="_csrf" th:content="${_csrf.token}">
    <meta name="_csrf_header" th:content="${_csrf.headerName}">
</th:block>
<script>
function csrfFetch(url, options) {
    var csrfMeta = document.querySelector('meta[name="_csrf"]');
    var headerMeta = document.querySelector('meta[name="_csrf_header"]');
    if (csrfMeta && headerMeta && csrfMeta.content && headerMeta.content) {
        options = options || {};
        options.headers = Object.assign({}, options.headers, {
            [headerMeta.content]: csrfMeta.content
        });
    }
    return fetch(url, options);
}
</script>
```

Key details per decisions:
- Per D-01: Use `<meta name="_csrf">` and `<meta name="_csrf_header">` with `th:content="${_csrf.token}"` / `${_csrf.headerName}`
- Per D-02: csrfFetch() is an inline `<script>` in the layout fragment, directly below the meta tags
- Per D-04: Graceful fallback — `th:if="${_csrf != null}"` prevents meta tags from rendering in dev/local. csrfFetch() additionally checks `csrfMeta.content && headerMeta.content` are truthy before injecting the header. If meta tags are absent (dev/local), csrfFetch() falls through to plain `fetch(url, options)`.
- Uses `Object.assign({}, options.headers, { ... })` to merge CSRF header without overwriting existing Content-Type header (per Pitfall 4 from RESEARCH.md).
- csrfFetch() is a pure function definition in `<head>` — no top-level DOM access (per Pitfall 5 from RESEARCH.md). Meta tag queries execute only when csrfFetch() is called at runtime.
