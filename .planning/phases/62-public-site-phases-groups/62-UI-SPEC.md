## Color

All values from `static/site/css/style.css` `:root` block.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (`--bg`) | Page background, body surface |
| Secondary (30%) | `#111` (`--bg-card`) | Tables (`.table-wrap`), nav, subnav, cards |
| Accent (10%) | `#4fc3f7` (`--accent`) | Reserved list below |
| Destructive | n/a | No destructive actions on the public static site |

Accent (`#4fc3f7`) reserved for:
1. Active phase tab (`.subnav-link.active color` — already established pattern)
2. Active group sub-tab (same `.subnav-link.active` pattern applied to secondary row)
3. Entity links in standings tables (`.entity-link color` — existing)
4. Active nav-link highlight (`.nav-link-active` — existing)
5. Empty-state banner border/icon accent (new, single use per page)

Accent must NOT be used on: inactive tab labels, table header text, body copy, breadcrumb separators, or hover backgrounds (hover uses `rgba(79, 195, 247, 0.1)` — semi-transparent, not the full accent token).

Supporting tokens (not 60/30/10 but used for depth):
- `--bg-hover: #1a1a1a` — table row hover
- `--border: #2a2a2a` — all borders
- `--text: #e0e0e0` — primary readable text
- `--text-dim: #888` — secondary/dimmed text
- `--text-muted: #555` — metadata, `th` cells, section titles

Note on `--accent-secondary`: `--accent-secondary: #7c4dff` exists in `:root` (style.css line 25) and is used for decorative gradients only (`.hero h1::after`, `.tile-card::before`). Must not be used on any new Phase 62 elements — doing so would break the 60/30/10 ratio.

## New CSS Classes — Phase Tab Rows

These classes are new additions to `static/site/css/style.css`. Exact names are declared here as part of the design contract so executor and checker use consistent identifiers.

### Phase-Tab Row (Row 1)

```
.phase-tab-row       — wrapper; reuses .subnav layout shape (bg-card, border-bottom, padding: 0 32px)
.phase-tab-row-inner — flex container; reuses .subnav-inner dimensions (max-width: 1100px, height: 44px, gap: 4px)
.phase-tab           — single tab anchor; reuses .subnav-link styles exactly
.phase-tab.active    — active state; reuses .subnav-link.active styles exactly
```

Visibility rule (server-side flag `showPhaseTabs`): rendered only when season has ≥2 phases. Single-REGULAR-LEAGUE seasons receive `showPhaseTabs = false` → no `.phase-tab-row` in output HTML → byte-identical to today.

### Group Sub-Tab Row (Row 2)

```
.group-tab-row       — wrapper; same shape as .phase-tab-row but height 36px (secondary, more compact)
.group-tab-row-inner — flex container; max-width: 1100px, height: 36px, gap: 4px
.group-tab           — single tab anchor; reuses .subnav-link styles exactly (font-size: 12px, same as .subnav-link)
.group-tab.active    — active state; same accent/background as .subnav-link.active
```

Visibility rule (server-side flag `showGroupTabs`): rendered only when current phase is GROUPS-layout. Combined view anchor is always first in the row (`href` = the phase's combined landing page, e.g. `standings.html` or `standings-regular.html`).

### Empty-State Banner

```
.empty-phase-banner  — full-width banner inside .section; border: 1px solid var(--border); border-radius: 8px;
                        background: var(--bg-card); padding: 24px; text-align: center
```

Banner appears inside `.section` above the standings table when phase has 0 race results. Table still renders with 0-point roster beneath it (D-22: predictable structure over silent skip).

## Tab Navigation Interaction Contract

Decision locked in CONTEXT.md D-07: **each phase variant is its own static HTML file**. No JS toggle, no `:target` selectors.

On sub-pages with no hero (standings, matchdays, driver-ranking pages), the phase-tab row is the primary visual anchor; the rank column in the standings table is the secondary anchor.

### Mechanism: Inter-file anchor links

- Each `.phase-tab` `<a>` element links to a sibling static file via relative path.
- Example: on `standings.html` (REGULAR combined), the PLAYOFF tab `<a href="playoff.html">` links to the existing bracket page.
- Example: on `standings-regular.html`, a group sub-tab links to `standings-regular-group-a.html`.
- Active tab is determined server-side at generation time. The generator writes `class="phase-tab active"` on the tab that corresponds to the current file. No client-side JS reads `window.location`.

### Static HTML one-URL-per-state invariant

Every distinct visible state has a distinct generated URL:
- `/season/{slug}/standings.html` — REGULAR combined (legacy default)
- `/season/{slug}/standings-regular.html` — REGULAR combined (explicit per-phase canonical)
- `/season/{slug}/standings-regular-group-{groupSlug}.html` — per-group view
- `/season/{slug}/standings-placement.html` — PLACEMENT phase (if exists)
- `/season/{slug}/matchdays.html` — REGULAR default
- `/season/{slug}/matchdays-{phaseSlug}.html` — per-phase
- `/season/{slug}/driver-ranking.html` — cross-phase aggregated default
- `/season/{slug}/driver-ranking-{phaseSlug}.html` — per-phase

The PLAYOFF tab always links to `/season/{slug}/playoff.html` (existing bracket page — D-08).

### Keyboard / accessibility contract (D-26)

On the phase-tab row container:
```html
<nav class="phase-tab-row" role="tablist" aria-label="Phase navigation">
```

On each tab anchor:
```html
<a class="phase-tab [active]"
   role="tab"
   aria-selected="true|false"
   href="standings-regular.html">Regular</a>
```

`aria-controls` points at the `id` of the page's `<section id="main-standings">` (or equivalent per-page section id). Keyboard navigation uses native Tab-key focus + Enter activation (anchor links). No JS arrow-key handler (D-26 deferred). Skip-link continues to work as today.

On the group sub-tab row container:
```html
<nav class="group-tab-row" role="tablist" aria-label="Group navigation">
```

Same `role="tab"` + `aria-selected` pattern on each group anchor.

## Standings Table — Group Column Contract

Controlled by server-side flag `showGroupColumn` (maps to CONTEXT.md D-32):

| Context | `showGroupColumn` | `showBuchholzColumn` |
|---------|-------------------|----------------------|
| LEAGUE-layout phase (all existing data) | false | false |
| GROUPS-layout, Combined View | true | false |
| GROUPS-layout, per-group view, non-Swiss format | false | false |
| GROUPS-layout, per-group view, Swiss format | false | true |

`showGroupColumn` and `showBuchholzColumn` are booleans passed by the page generator into the Thymeleaf model. Templates use `th:if="${showGroupColumn}"` on the `<th>Group</th>` column and corresponding `<td>` cells. No SpEL projections in templates (CLAUDE.md "Keep Templates Lean").

Column order in standings table when all flags true (Combined View, Swiss format — hypothetical max):
`# | Team | Group | MP | W | D | L | Buchholz | Pts`

Column order in normal LEAGUE view (today's format, unchanged):
`# | Team | MP | W | D | L | PR | Pts`

Column order in GROUPS Combined View (Group column visible, Buchholz hidden):
`# | Team | Group | MP | W | D | L | PR | Pts`

## Profile Pages — Phase Breakdown Section Contract

### team-profile.html (D-13/D-14)

Controlled by server-side flag `showPhaseBreakdown` (true when season has ≥2 phases):

When `showPhaseBreakdown = false`: page renders identically to today (SC4 invariant).

When `showPhaseBreakdown = true`: new section appears after the main standings panel:
```
Section title: "Phase Breakdown"   (.section-title Conthrax 13px uppercase)
Content:       one row per phase the team participated in:
               "<Phase label>: <rank><ordinal> place, <points> pts"
               e.g. "Regular: 5th place, 47 pts | Playoff: Semifinal exit"
```

The main standings panel continues to use combined-view standing (`calculateStandings(REGULAR, null)`) per D-14.

### driver-profile.html (D-15)

Controlled by server-side flag `showPhaseBreakdown` (true when season has ≥2 phases):

When `showPhaseBreakdown = false`: page renders identically to today (SC4 invariant).

When `showPhaseBreakdown = true`: results section gains per-phase headings:
- H3 heading per phase: "Regular Season Results" / "Playoff Results" / "Placement Phase Results"
- Same Conthrax 13px uppercase style as `.section-title`

## Responsive Behavior Contract

### Phase-tab row (`.phase-tab-row`)

Desktop: horizontal flex row, height 44px, tabs visible inline. Matches `.subnav` desktop behavior.

Mobile (`max-width: 768px`): horizontal overflow scroll — same pattern as `.subnav` mobile rule in `style.css`:
```css
@media (max-width: 768px) {
    .phase-tab-row { padding: 0 16px; overflow-x: auto; -webkit-overflow-scrolling: touch; }
    .phase-tab-row-inner { height: auto; padding: 8px 0; flex-wrap: nowrap; }
}
```

No dropdown/accordion fallback for mobile (deferred per CONTEXT.md deferred section). Production data has 1–2 phases per season — horizontal scroll is sufficient.

### Group sub-tab row (`.group-tab-row`)

Same overflow-scroll pattern as phase-tab row on mobile. Height reduces from 36px to auto on mobile.

### Standings table

Existing `.table-wrap` overflow-x behavior covers the added Group/Buchholz columns. The right-fade gradient overlay (`@media (max-width: 768px) .table-wrap::after`) already signals scrollability. No additional mobile treatment needed.

### Profile phase breakdown section

Stacks vertically on all screen sizes. Standard `.section` margin and padding applies.

## Static Generation Contract

### Files generated per LEAGUE-layout single-REGULAR season (SC4 invariant)

```
/season/{slug}/standings.html          ← REGULAR combined (no tab row)
/season/{slug}/matchdays.html          ← REGULAR matchdays (no tab row)
/season/{slug}/driver-ranking.html     ← cross-phase aggregated (no tab row)
```

Identical to today. Tab rows absent. `showPhaseTabs = false`, `showGroupTabs = false`.

### Files generated per GROUPS-layout single-REGULAR season

```
/season/{slug}/standings.html          ← REGULAR combined (group sub-tab row visible, no phase tab row)
/season/{slug}/standings-regular-group-{groupSlug}.html  ← per each group
/season/{slug}/matchdays.html          ← REGULAR combined matchdays
/season/{slug}/matchdays-regular-group-{groupSlug}.html  ← per each group (if applicable)
/season/{slug}/driver-ranking.html     ← cross-phase aggregated
/season/{slug}/driver-ranking-regular.html  ← per-REGULAR phase
```

`showPhaseTabs = false` (only 1 phase), `showGroupTabs = true`.

### Files generated per multi-phase season (e.g. REGULAR + PLAYOFF)

```
/season/{slug}/standings.html          ← REGULAR combined (phase tab row visible)
/season/{slug}/standings-regular.html  ← REGULAR combined explicit (same content as legacy URL)
/season/{slug}/matchdays.html          ← REGULAR default
/season/{slug}/matchdays-regular.html  ← REGULAR per-phase explicit
/season/{slug}/matchdays-playoff.html  ← PLAYOFF per-phase
/season/{slug}/driver-ranking.html     ← cross-phase aggregated default
/season/{slug}/driver-ranking-regular.html   ← per-REGULAR
/season/{slug}/driver-ranking-playoff.html   ← per-PLAYOFF (if PLAYOFF has driver data)
```

PLAYOFF tab in the phase tab row links to existing `/season/{slug}/playoff.html` (not a generated `standings-playoff.html` per D-08).

`showPhaseTabs = true`, `showGroupTabs = false` (assuming REGULAR is LEAGUE-layout).
