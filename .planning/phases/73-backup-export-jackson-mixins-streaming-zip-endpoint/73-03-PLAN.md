<objective>
Ship the service layer that turns Plan 73-01's MixIns + Plan 73-02's repositories into a streamed ZIP. Two services with clean separation of concerns:

- `BackupExportService` (`@Service`, `@Transactional(readOnly=true)`) — the DB-read orchestrator. Three public methods: `countRowsPerTable()` (computes the manifest's `tableCounts` upfront), `fetchAllForBackup(Class<?>)` (dispatches to the right `findAllForBackup()` repository call), and `enumerateReferencedUploads()` (walks Team/SeasonTeam/Car/Track/RaceAttachment for upload-path references and returns a deduplicated, on-disk-verified list).
- `BackupArchiveService` (`@Service`) — stateless ZIP plumbing. One public method `writeZip(OutputStream out, Instant exportDate)`: opens `ZipOutputStream`, writes `manifest.json` as entry #0, iterates `BackupSchema.getExportOrder()` writing `data/<slug>.json` per entity, then writes `uploads/<rel>` for every `UploadEntry`.

Plus 5 test files: 2 Surefire unit tests (Mockito-driven service unit tests) and 4 Failsafe ITs (`BackupExportServiceIT` for fixture-backed export shape, `BackupArchiveServiceIT` for manifest-first invariant, `BackupExportNoLazyInitIT` for the LIE-absence guard on Saison-2023+2024-3, `BackupUploadsMirrorIT` for the uploads mirror, `BackupRoundTripIT` for the manifest round-trip).

Purpose: EXPORT-02 (streaming, no full-dataset memory buffering), EXPORT-03 (manifest-first + data/<entity>.json + uploads/ mirror), EXPORT-05 (`@Transactional(readOnly=true)` + no `LazyInitializationException`).

Output: 3 new production files + 5 new test files under `src/main/java/org/ctc/backup/service/` and `src/test/java/org/ctc/backup/service/`.
</objective>
