## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Service → Template | Pre-computed URLs pass from Java service to Thymeleaf template context |
| User browser → Static HTML | Users click `<a>` links in static HTML; all URLs are relative paths |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-02 | S (Spoofing) | Entity link URLs | accept | URLs are pre-computed from DB slugs server-side; no user input in URL construction; static HTML has no runtime URL modification |
| T-39-03 | T (Tampering) | URL slug injection | mitigate | `slugify()` method strips all non-alphanumeric characters and normalizes to lowercase; no special chars can survive into href attributes |
| T-39-04 | I (Information Disclosure) | Driver/team enumeration | accept | Static site is public by design; all team and driver data is intentionally visible; no PII beyond PSN-IDs which are public gaming identifiers |
| T-39-05 | D (Denial of Service) | N+1 query in generateTeamProfiles | accept | `findByDriverId()` is called per driver per team; acceptable for static site generation (runs once, not per-request); bounded by number of teams x drivers |
</threat_model>
