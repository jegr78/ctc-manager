<objective>
Introduce a single central team-size constant and make the Lineup and Results
graphics render a fixed 6 rows per side, with missing driver slots shown as the
literal string "n/a" (Results also shows 0 points for the missing slot). The fix
lives entirely in the service render-DTO builders and their templates' display
guards — no business logic is added to templates or controllers.

Purpose: Closes LINEUP-01 (Lineup n/a) and LINEUP-02 (Results n/a + 0 points),
and establishes the `TEAM_DRIVERS` source of truth that plan 108-02 reuses.

Output: `AbstractGraphicService.TEAM_DRIVERS`, padded `buildPairings` /
`buildResultRows`, updated template guards, updated unit tests.
</objective>

ResultsGraphicService.DriverResultRow record:
  record DriverResultRow(String homeDriver, String homeNickname, int homePoints, int awayPoints, String awayDriver, String awayNickname)
Current buildResultRows (~L164) computes `int maxSize = Math.max(homeResults.size(), awayResults.size())`
and fills absent slots with "" (name/nick) and 0 (points).

lineup-render.html row block (~L90-98) — pairing display:

  (mirror for away)

results-render.html row block (~L94-104) — result display:

  (mirror for away)
