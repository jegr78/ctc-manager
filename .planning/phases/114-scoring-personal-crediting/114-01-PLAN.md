<objective>
Establish the Wave-0 foundation for Phase 114: a guest scenario in the shared `@Profile("dev")` seed (`TestDataService.seedRaceLineups`) covering both a **doppelrollen guest** (roster Team A + guest for Team B) and a **pure guest** (no `SeasonDriver`, guest lineup only), each with `RaceResult` rows and an aggregated match score (D-11). Wire `DevDataSeeder` (`@Profile("dev")`) to verify and log the seeded guest example so the `dev,demo` profile renders real guest data for `/gsd-auto-uat 114` and the Phase 115 visual reference (D-12). Also create the empty regression IT skeleton (`DriverRankingServiceGuestIT`) so downstream plans implement against a known contract.

This plan is the prerequisite for the unify refactor (Plan 02), the profile-page hook (Plan 03), and the regression IT suite (Plan 04). It contains NO production scoring logic changes — only test/seed data, a dev-only seeder verification hook, and an empty test scaffold.

Architecture note (why D-12 lives where it does): the `dev,demo` profile's domain data (seasons, teams, drivers, lineups, results) is produced ENTIRELY by `TestDataService.seed()` → `seedRaceLineups()`, invoked by `DevDataSeeder` (`@Profile("dev")`). The separate `DemoDataSeeder` (`@Profile("demo")`) only imports GT7 cars/tracks/images. Therefore the guest `new RaceLineup(..., true)` rows belong in `TestDataService` (Task 1, also satisfies D-11) — duplicating them into the thin `DevDataSeeder` CommandLineRunner would violate CLAUDE.md "Keep Controllers/Runners Thin" and "No Fallback Calculations". D-12 is satisfied by giving `DevDataSeeder` an explicit post-seed guest-verification + log hook (Task 2) that fails loudly if the `dev,demo` startup seed is missing the guest example.

Purpose: Without a guest in the seed, none of the SCORE-01/02/03 integration tests or the `/gsd-auto-uat 114` render smoke have real data to exercise.
Output: Extended `TestDataService.seedRaceLineups`; `DevDataSeeder` guest-verification hook; new empty `DriverRankingServiceGuestIT.java`.
</objective>
