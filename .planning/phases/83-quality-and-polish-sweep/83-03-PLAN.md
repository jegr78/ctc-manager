<objective>
Implement QUAL-03: Per-group matchday generation UI affordance for GROUPS-layout seasons.

Purpose: `SeasonController.java:251` currently hardcodes `groupId=null` when calling `matchdayGeneratorService.generate(...)`, which means GROUPS-layout seasons cannot be processed via the admin UI at all (the service throws `IllegalArgumentException("GROUPS layout requires non-null groupId")` at `MatchdayGeneratorService.java:49-51`). This plan adds a conditional `<select>` to `matchday-generator.html` for GROUPS-layout phases, threads the choice through `MatchdayGeneratorForm`, and wires it to the existing service parameter — verified by a Playwright E2E on Season 2023.

Output: Form DTO + template + controller change in one feature commit; Playwright E2E in a second test commit (per D-02 default split, TDD-shape allows merging).
</objective>
