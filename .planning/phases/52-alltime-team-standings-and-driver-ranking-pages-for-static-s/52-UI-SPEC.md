## Typography

| Role                          | Size  | Weight      | Line Height              |
| ----------------------------- | ----- | ----------- | ------------------------ |
| Body / table cell             | 14px  | 400 (regular) | 1.6 (inherited from body) |
| Label / table header          | 11px  | 400 (regular) | inherited                |
| Section title (Conthrax)      | 13px  | 700 (bold)  | inherited                |
| Hero / page heading (Conthrax) | 28px | 700 (bold)  | inherited                |

Weights: 2 only ‚Äî 400 (regular) and 700 (bold). No other weights are part of this contract.

Notes:

- Hero `h1` uses `font-family: 'Conthrax', sans-serif`, `font-size: 28px`, `letter-spacing: 3px`, `font-weight: 700`. Source: `.hero h1` in `style.css`.
- Section titles use `font-family: 'Conthrax', sans-serif`, `font-size: 13px`, `text-transform: uppercase`, `letter-spacing: 2px`, `font-weight: 700`. Source: `.section-title`.
- Table headers `th`: `font-size: 11px`, `text-transform: uppercase`, `letter-spacing: 1px`, `color: var(--text-muted)`, `font-weight: 400`. Source: `th` ruleset.
- Table body `td`: `font-size: 14px`. Source: `table` ruleset.
- Rank number and points columns use `.font-bold .text-white .text-lg` utility classes ‚Äî these apply `font-weight: 700` and `font-size: 16px`. The 16px value is an **inherited pre-existing CSS utility** (`.text-lg`) not a new size declared by this phase. Same applies to `.hero-sub` which sets `font-size: 15px` ‚Äî also an inherited pre-existing utility, not part of the declared 4-size scale above.
- Mobile hero: `font-size: 22px` at `max-width: 768px`. Source: responsive block. This is a responsive override of the 28px heading size, not a separate scale entry.

Phase 52 introduces no new type sizes and no new weights. All values are pre-populated from the existing CSS.

## Interaction States

### Navigation active state

The alltime pages register as top-level nav items. `currentPage` values:

- `alltime-standings` ‚Üí "Standings" nav link gets `.nav-link-active`
- `alltime-driver-ranking` ‚Üí "Driver Ranking" nav link gets `.nav-link-active`

No `seasonSlug` is set for either page ‚Üí subnav is hidden (existing layout `th:if="${seasonSlug != null ...}"` evaluates to false).

Source: 52-CONTEXT.md decisions + layout.html subnav condition.

### Table row hover

`tr:hover { background: var(--bg-hover); }` ‚Äî inherited, no changes needed.
Transition: `transition: background-color 0.2s` on `tr`. Source: existing `style.css`.

### Mobile scroll indicator

`.table-wrap::after` fade gradient is active at `max-width: 768px` ‚Äî inherited from existing CSS, no new work required.

## Implementation Notes for Executor

1. **No new CSS classes needed.** All visual elements reuse existing `style.css` classes: `.section`, `.section-title`, `.hero-sub`, `.table-wrap`, `table`, `th`, `td`, `.font-bold`, `.text-white`, `.text-center`, `.text-right`, `.text-dim`, `.text-lg`.

2. **Subtitle element.** Use `<p class="hero-sub">All seasons combined</p>` directly below the `<h2 class="section-title">` ‚Äî same pattern as season pages use `.season-meta` for year/number. The `hero-sub` class (`color: var(--text-dim); font-size: 15px; margin-top: 8px`) renders the correct muted subtitle style.

3. **No `seasonSlug` in context.** Do not set `seasonSlug` in the Thymeleaf context for either alltime page. This suppresses the subnav and collapses the breadcrumb to `Home > {page}`.

4. **Navigation replacement.** In `layout.html`, replace the two `th:if="${activeSeasonSlug != null ...}"` conditional nav links with unconditional alltime links. The existing `.nav-link-active` class mechanism continues to work via `currentPage` matching.

5. **Table structure is a direct copy of the season analog**, minus the `entity-link` wrapper on names. The rank `#` column and `Pts` column styling are identical.
