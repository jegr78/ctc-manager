## Page Structure

### alltime-standings.html

```
layout fragment: layout('Alltime Standings', ...)
├── [inherited] skip-link
├── [inherited] nav (.nav) — "Standings" link active (alltime-standings)
├── [NO subnav] — no seasonSlug context variable set
├── breadcrumb: Home > Alltime Standings
└── main.main#main-content
    └── div.section
        ├── h2.section-title — "Alltime Team Standings"
        ├── p.hero-sub — "All seasons combined"
        └── div.table-wrap
            └── table
                ├── thead: # | Team | MP | W | D | L | PR | Pts
                └── tbody: one tr per standing entry
```

Column alignment per existing standings.html pattern:
- `#` — left-aligned, `.font-bold .text-white`
- `Team` — left-aligned, plain text (no entity-link)
- `MP` — `.text-center`
- `W` — `.text-center`
- `D` — `.text-center`
- `L` — `.text-center`
- `PR` — `.text-center .text-dim`
- `Pts` — `.text-right .font-bold .text-white .text-lg`

### alltime-driver-ranking.html

```
layout fragment: layout('Alltime Driver Ranking', ...)
├── [inherited] skip-link
├── [inherited] nav (.nav) — "Driver Ranking" link active (alltime-driver-ranking)
├── [NO subnav] — no seasonSlug set
├── breadcrumb: Home > Alltime Driver Ranking
└── main.main#main-content
    └── div.section
        ├── h2.section-title — "Alltime Driver Ranking"
```

Column alignment per existing driver-ranking.html pattern:

Source: driver-ranking.html existing template pattern + 52-CONTEXT.md column spec.

## Interaction States

### Navigation active state

The alltime pages register as top-level nav items. `currentPage` values:
- `alltime-standings` → "Standings" nav link gets `.nav-link-active`
- `alltime-driver-ranking` → "Driver Ranking" nav link gets `.nav-link-active`

No `seasonSlug` is set for either page → subnav is hidden (existing layout `th:if="${seasonSlug != null ...}"` evaluates to false).

Source: 52-CONTEXT.md decisions + layout.html subnav condition.

### Table row hover

`tr:hover { background: var(--bg-hover); }` — inherited, no changes needed.
Transition: `transition: background-color 0.2s` on `tr`. Source: existing `style.css`.

### Mobile scroll indicator

`.table-wrap::after` fade gradient is active at `max-width: 768px` — inherited from existing CSS, no new work required.

## Implementation Notes for Executor

1. **No new CSS classes needed.** All visual elements reuse existing `style.css` classes: `.section`, `.section-title`, `.hero-sub`, `.table-wrap`, `table`, `th`, `td`, `.font-bold`, `.text-white`, `.text-center`, `.text-right`, `.text-dim`, `.text-lg`.

2. **Subtitle element.** Use `<p class="hero-sub">All seasons combined</p>` directly below the `<h2 class="section-title">` — same pattern as season pages use `.season-meta` for year/number. The `hero-sub` class (`color: var(--text-dim); font-size: 15px; margin-top: 8px`) renders the correct muted subtitle style.

3. **No `seasonSlug` in context.** Do not set `seasonSlug` in the Thymeleaf context for either alltime page. This suppresses the subnav and collapses the breadcrumb to `Home > {page}`.

4. **Navigation replacement.** In `layout.html`, replace the two `th:if="${activeSeasonSlug != null ...}"` conditional nav links with unconditional alltime links. The existing `.nav-link-active` class mechanism continues to work via `currentPage` matching.

5. **Table structure is a direct copy of the season analog**, minus the `entity-link` wrapper on names. The rank `#` column and `Pts` column styling are identical.
