## Template Binding Change Contract

This section is the primary design contract for Phase 29. It specifies the exact attribute changes required in `matchday-form.html`.

### Before (current — mass assignment vulnerability)

```html
<form th:object="${matchday}">
    <input type="hidden" th:field="*{id}">
    <input type="hidden" name="seasonId" th:value="${matchday.season != null ? matchday.season.id : ''}">
    <div th:if="${matchday.season == null}">...</div>
    <div th:if="${matchday.season != null}">
        <strong th:text="${matchday.season.displayLabel}"></strong>
    </div>
    <h1 th:text="${matchday.id != null ? 'Edit Matchday' : 'New Matchday'}"></h1>
```

### After (required — DTO binding)

```html
<form th:object="${form}">
    <input type="hidden" th:field="*{id}">
    <input type="hidden" th:field="*{seasonId}" th:if="${form.seasonId != null}">
    <div class="form-group" th:if="${form.seasonId == null}">...</div>
    <div th:if="${season != null}" class="text-dim mb-md">
        Season: <strong th:text="${season.displayLabel}"></strong>
    </div>
    <h1 th:text="${form.id != null ? 'Edit Matchday' : 'New Matchday'}"></h1>
```

### Binding rules

| Template expression | Before | After | Reason |
|--------------------|--------|-------|--------|
| `th:object` | `${matchday}` | `${form}` | D-06: bind to MatchdayForm, not JPA entity |
| Season hidden field | `name="seasonId" th:value="${matchday.season...}"` | `th:field="*{seasonId}"` | D-02: seasonId lives on DTO; th:field is canonical pattern |
| Season select visibility | `th:if="${matchday.season == null}"` | `th:if="${form.seasonId == null}"` | MatchdayForm has no `season` property — check UUID field directly |
| Season display | `th:if="${matchday.season != null}"` → `matchday.season.displayLabel` | `th:if="${season != null}"` → `${season.displayLabel}` | D-07: season entity loaded separately into model as `${season}` |
| Page heading condition | `${matchday.id != null}` | `${form.id != null}` | `matchday` removed from model; use `form` |

### Model attributes per flow

| Flow | Model attributes required |
|------|--------------------------|
| GET /new (no seasonId param) | `form` (empty MatchdayForm), `seasons` (list) |
| GET /new?seasonId=X | `form` (MatchdayForm with seasonId set), `season` (Season entity), `seasons` (list) |
| GET /{id}/edit | `form` (MatchdayForm mapped from entity), `season` (Season entity), `seasons` (list) |
| POST /save — validation error | `seasons` (list), `season` (if form.seasonId != null) — form repopulated by Spring MVC |
| POST /save — success | redirect to /admin/matchdays?seasonId={id} with `successMessage` flash attribute |
