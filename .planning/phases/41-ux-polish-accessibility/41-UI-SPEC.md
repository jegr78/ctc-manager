## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, skip-link padding |
| md | 16px | Default element spacing, cell padding |
| lg | 24px | Section padding, `.driver-header` margin-bottom (replaces inline style line 6) |
| xl | 32px | Layout gaps, main padding |
| 2xl | 48px | Footer top margin, hero padding |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Nav height: 60px (not on scale — do not change)
- Subnav height: 44px (established touch target — do not change)
- `.match-team` font-size 18px — intentional display size outside scale

Source: Existing `style.css` measurements confirmed.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.6 |
| Label / meta | 11–12px | 400 (regular) | 1.4 |
| Heading (section) | 13px | 600 (Conthrax) | 1.2 |
| Display (hero h1) | 28px / 22px mobile | 600 (Conthrax) | 1.2 |

Rules:
- Active contract uses 2 weights: 400 (regular body) and 600 (Conthrax headings, bold cells). All new work in this phase uses only these weights.
- Skip-link: 14px at weight 400, matches body.
- Footer links: 13px at weight 400, matches existing `.footer` font-size.

Legacy exception:
- `font-weight: 700` — grandfathered in `.match-team` and `.font-bold` utility class only. These pre-exist from Phase 37 and remain unchanged. No new usage of weight 700 in this phase.

Source: `style.css` — `body` (line-height 1.6), `table` (font-size 14px), `th` (11px), `.section-title` (13px Conthrax), `.hero h1` (28px Conthrax).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` (`--bg`) | Page background |
| Secondary (30%) | `#111` (`--bg-card`) | Cards, nav bar, subnav, footer background, table-wrap |
| Accent (10%) | `#4fc3f7` (`--accent`) | Reserved list below |
| Destructive | n/a | No destructive actions in this phase |

**Accent (`#4fc3f7`) reserved for:**
1. `.match-team-winner` — winning team name text color
2. `.nav-link-active` background tint `rgba(79,195,247,0.1)` — active nav item
3. `.subnav-link.active` — active subnav item
4. `.entity-link` — cross-linking anchor elements
5. `.match-score` — match score display
6. `.footer-link:hover` — footer link hover state only (dim at rest, accent on hover)
7. `.breadcrumb-link:hover` — breadcrumb hover (existing pattern)
8. `.skip-link` — skip-link background on focus (high-contrast: white text on accent background)
9. `.bracket-team.winner` — bracket winner (existing)

**Winner highlight background:**
- `.match-team-winner` background: `rgba(79,195,247,0.15)` semi-transparent accent
- Border-radius: `var(--radius-sm)` (6px)
- Padding: 4px 8px (xs/sm on-scale values)

Source: CONTEXT.md D-01; existing `--accent` usage in `style.css`.

**Hover states:**
- Table rows: `--bg-hover` (`#1a1a1a`) — existing `tr:hover` rule, add `transition: background-color 0.2s`
- Nav links: existing `rgba(255,255,255,0.05)` background — do not change
- Footer links: dim at rest (`--text-dim` `#888`), accent on hover (`--accent`) — matches `.breadcrumb-link` pattern

## Interaction Contracts

### Skip-to-Content Link (UX-01)

| Property | Value |
|----------|-------|
| Position in DOM | First child of `<body>` in `layout.html` |
| Target anchor | `id="main-content"` on `<main class="main">` |
| Resting state | Visually hidden: `position: absolute; left: -9999px; top: auto; width: 1px; height: 1px; overflow: hidden` |
| Focus state | Slides into view: `position: fixed; top: 8px; left: 8px; z-index: 9999; padding: 8px 16px; background: var(--accent); color: #000; border-radius: var(--radius-sm); font-size: 14px; font-weight: 600; text-decoration: none` |
| Copy | "Skip to main content" |
| Trigger | `:focus` pseudo-class only |

Source: CONTEXT.md D-09; standard WCAG 2.1 skip-link pattern.

### Nav Toggle Aria-Label (UX-07)

| Property | Value |
|----------|-------|
| Remove from | `<input type="checkbox" ... aria-label="...">` — aria-label on hidden input is ignored by AT |
| Add to | `<label for="nav-toggle" class="nav-toggle-label">` |
| `aria-label` value | "Toggle navigation menu" |
| `role` | `role="button"` added to `<label>` |
| `aria-expanded` | Optional: add `aria-expanded` via CSS `content` trick or omit if pure CSS; Claude's Discretion per CONTEXT.md |

Source: CONTEXT.md D-10.

### Match Winner Highlight (UX-04)

| Property | Value |
|----------|-------|
| CSS class | `.match-team-winner` |
| Applied via | `th:class` conditional in `matchday.html` and `index.html` |
| Condition | Pre-computed `race.homeTeamWon` / `race.awayTeamWon` boolean fields on `RaceView` |
| Text color | `var(--accent)` |
| Background | `rgba(79,195,247,0.15)` |
| Padding | 4px 8px |
| Border-radius | `var(--radius-sm)` (6px) |
| No transition | Winner highlight is static — no animation |

Source: CONTEXT.md D-01, D-02; existing `.bracket-team.winner` pattern at `style.css` line 352.

### Mobile Scroll Indicator (UX-05)

| Property | Value |
|----------|-------|
| Target element | `.table-wrap` |
| Implementation | `::after` pseudo-element |
| Gradient | `linear-gradient(to right, transparent, var(--bg-card))` |
| Width | 40px |
| Height | 100% |
| Position | `absolute; right: 0; top: 0; pointer-events: none` |
| Media query | `@media (max-width: 768px)` only |
| Parent positioning | `.table-wrap` needs `position: relative` |
| Static fade | Acceptable per D-04; no JS scroll detection required |

Source: CONTEXT.md D-03, D-04.

### Footer Links (UX-06)

| Property | Value |
|----------|-------|
| Layout | Two rows: link bar above, branding below |
| Link bar items | "Top" (smooth-scroll `href="#top"`), "Archive" (`href="${rootPath}/archive.html"`), active season name (`href="${rootPath}/season/${activeSeasonSlug}/standings.html"`) |
| Separator | centered dot `·` (U+00B7) |
| CSS class | `.footer-link` |
| Resting color | `var(--text-dim)` (`#888`) |
| Hover color | `var(--accent)` |
| Transition | `color 0.2s` |
| Font-size | 13px (matches existing `.footer`) |
| Conditional | Active season link only rendered when `activeSeasonSlug != null` |
| Smooth scroll | `html { scroll-behavior: smooth }` added globally |

Source: CONTEXT.md D-05, D-06.

### Hover Transitions (UX-08)

| Element | Property | Duration |
|---------|----------|----------|
| `tr` | `background-color` | 0.2s |
| `.footer-link` | `color` | 0.2s |
| `.skip-link` | none (appears/disappears on focus) | — |

Do not modify existing transitions on `.nav-links a` (all 0.2s), `.entity-link` (all 0.2s), `.subnav-link` (all 0.2s), `.breadcrumb-link` (color 0.2s).

Source: CONTEXT.md D-07; REQUIREMENTS.md UX-08 ("150-300ms" — 0.2s = 200ms is within range).

### cursor:pointer (UX-09)

| Selector | Reason |
|----------|--------|
| `a` | All links |
| `label[for]` | Covers nav toggle label |
| `[role="button"]` | Future-proof for any role=button elements |

Single global rule block, added once in `style.css`. `.match-card` does not get `cursor: pointer` unless it has a wrapping `<a>` — Claude's Discretion per CONTEXT.md.

Source: CONTEXT.md D-08.

### Inline Style Removal (QUAL-01)

| Location | Inline style | Replacement class |
|----------|-------------|-------------------|
| `driver-profile.html` line 6 | `style="margin-bottom: 24px;"` | `.driver-header` |
| `driver-profile.html` line 47 | `style="margin-top: 24px;"` | `.section-gap` |

CSS definitions:
```css
.driver-header { margin-bottom: 24px; }   /* lg spacing token */
.section-gap   { margin-top: 24px; }       /* lg spacing token */
```

Source: CONTEXT.md D-11.
