    Plans 01-05 have been merged to master via PR (operator-managed per CLAUDE.md Git Workflow). The ci.yml workflow has run 1+ times against the merged code. Now the operator needs to ensure 5 consecutive CI runs exist for the median computation. Per RESEARCH §Pattern 5, GitHub Actions does not natively loop a single step 5 times — the practical approach is `workflow_dispatch` manual re-trigger or relying on natural PR/push runs.
  </what-built>
  <how-to-verify>
    Step 1 — confirm at least 5 consecutive ci.yml runs exist on master after the Plan-05 merge:

    ```bash
    gh run list --workflow=ci.yml --branch=master --limit=10 \
      --json conclusion,createdAt,databaseId,headSha,name,status,event
    ```

    Expect ≥5 `conclusion=success` runs newer than the Plan-05 merge commit. If fewer than 5 successful runs exist, trigger more via `workflow_dispatch` (if the workflow supports it) or via empty commits that trigger CI:

    ```bash
    # If workflow_dispatch is enabled:
    gh workflow run ci.yml --ref master

    # Otherwise via empty commits (operator only — never automate):
    git commit --allow-empty -m "ci(86): trigger CI rerun for PERF-05 harvest" && git push origin master
    ```

    Step 2 — extract Maven Total time from each of the 5 most recent successful runs:

    ```bash
    for id in $(gh run list --workflow=ci.yml --branch=master --status=success --limit=5 --json databaseId --jq '.[].databaseId'); do
      echo "=== Run $id ==="
      gh run view "$id" --log 2>&1 | grep -E "BUILD SUCCESS|\[INFO\] Total time:" | tail -5
    done
    ```

    Record the 5 Maven "Total time" values. Drop min and max. Compute median of the 3 middle values.

    Step 3 — variance check (D-10): if (max − min) / median > 20%, repeat Step 2 with the next 5 runs (newer ones) or trigger 5 fresh ones.
  </how-to-verify>
