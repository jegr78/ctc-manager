## Root Cause Analysis

**Same pattern, opposite direction.** The first audit found Phase 23 overwrote Phase 22's fictive data. Phase 24 was created to fix this by restoring Phase 22's state. But Phase 24 used Phase 22 (commit `5472fd6`) as its source — which predates Phase 23-02 entirely. This means Phase 24 silently removed Phase 23-02's entire matchday/result/scoring pipeline.

The fundamental problem: **restoring one phase's work by reverting to a prior state destroys all intermediate phases' contributions to the same file.** A proper fix must merge all phases' changes, not replace one with another.
